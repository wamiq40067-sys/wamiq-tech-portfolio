# 1. More About Frameworks and Controls

- Security frameworks are structured guidelines that help organizations manage cybersecurity risks and protect sensitive data.
- Frameworks support compliance with laws, regulations, and industry standards.
- Security controls are safeguards implemented to reduce risks, threats, and vulnerabilities.
- Controls are generally divided into three categories:
  - Physical controls
  - Technical controls
  - Administrative controls
- Physical controls include:
  - Locks
  - CCTV cameras
  - Security guards
  - Access cards
- Technical controls include:
  - Firewalls
  - Antivirus software
  - Multi-factor authentication (MFA)
- Administrative controls include:
  - Authorization policies
  - Asset classification
  - Separation of duties
- Common cybersecurity frameworks include:
  - NIST Cybersecurity Framework (CSF)
  - ISO/IEC 27001
- Frameworks and controls work together to strengthen an organization’s security posture.

# 2. The CIA Triad: Confidentiality, Integrity, and Availability

- The CIA triad is a core cybersecurity model used to manage security risks.
- CIA stands for:
  - Confidentiality
  - Integrity
  - Availability
- Confidentiality ensures that only authorized users can access sensitive information.
- The principle of least privilege helps maintain confidentiality by limiting unnecessary access.
- Integrity ensures data remains accurate, authentic, and reliable.
- Encryption and cryptography help maintain data integrity and confidentiality.
- Availability ensures systems and data are accessible to authorized users whenever needed.
- Backup systems, maintenance, and secure remote access help maintain availability.
- The CIA triad forms the foundation of many cybersecurity practices and policies.

# 3. NIST Frameworks

- The NIST Cybersecurity Framework (CSF) is a widely used cybersecurity framework developed by the National Institute of Standards and Technology.
- The framework helps organizations identify, assess, and manage cybersecurity risks.
- NIST CSF is voluntary and based on industry standards and best practices.
- The framework contains six core functions:
  - Govern
  - Identify
  - Protect
  - Detect
  - Respond
  - Recover
- Govern focuses on cybersecurity strategy, leadership, and risk management.
- Identify focuses on understanding assets, systems, and cybersecurity risks.
- Protect involves implementing safeguards and security controls.
- Detect focuses on identifying cybersecurity incidents quickly.
- Respond involves managing and containing security incidents.
- Recover focuses on restoring systems and operations after an attack or disruption.
- NIST SP 800-53 provides detailed security controls for federal information systems.

# 4. OWASP Principles and Security Audits

## OWASP Principles

- OWASP is a nonprofit organization focused on improving software and application security.
- Establish secure defaults means systems should remain secure using default settings.
- Fail securely means systems should move to the safest state if a failure occurs.
- Don’t trust services means organizations should verify third-party systems instead of assuming they are secure.
- Avoid security by obscurity means security should not depend only on secrecy or hidden system details.
- Strong security should rely on layered defenses, monitoring, secure passwords, and proper security controls.

## Security Audits

- A security audit is an independent review of an organization’s security policies, procedures, and controls.
- Audits help identify risks, vulnerabilities, and compliance gaps.
- Audits ensure organizations follow security standards and legal requirements.
- Security audits evaluate:
  - Administrative controls
  - Technical controls
  - Physical controls
- An audit checklist usually includes:
  - Identifying the audit scope
  - Performing a risk assessment
  - Conducting the audit
  - Creating a mitigation plan
  - Reporting findings to stakeholders
- Audits help improve an organization’s overall security posture.
