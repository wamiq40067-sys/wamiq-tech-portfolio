# Key Notes: Introduction to Operating Systems

## 1. The Wonderful World of Operating Systems

- An **operating system (OS)** is the main software that manages computer hardware and allows users to interact with the system.
- It acts as a bridge between **users, applications, and hardware**.
- Common operating systems include:
  - Windows
  - macOS
  - Linux
  - ChromeOS
  - Android
  - iOS
- Each OS has unique features, security models, and use cases.
- Operating systems are essential for:
  - Running applications
  - Managing memory and storage
  - Controlling hardware resources
- Security is a major concern in OS design:
  - Systems must be regularly updated
  - Vulnerabilities can exist in all OS types
- **Legacy operating systems** are outdated systems still in use, often due to compatibility needs, but they increase security risks.

---

## 2. The Operating System at Work

- The OS manages communication between **user requests and hardware actions**.
- Basic workflow:
  1. User initiates a task
  2. Application processes the request
  3. Operating system interprets and forwards the request
  4. Hardware executes the task
  5. Results are sent back to the user via the OS and application
- The OS performs key functions:
  - Memory management (RAM usage)
  - Process management (running programs)
  - File management (storage and retrieval)
  - Device management (hardware coordination)
- **Boot process:**
  - BIOS/UEFI initializes system hardware
  - Bootloader loads the operating system into memory
  - OS takes control of system operations
- Analogy:
  - OS is like a "manager" or "kitchen" that processes user requests behind the scenes

## 3. The User Interface

- A **user interface (UI)** allows users to interact with the operating system.

### Types of User Interfaces:

#### Graphical User Interface (GUI)
- Uses visual elements like:
  - Icons
  - Windows
  - Menus
- Easy to use and beginner-friendly
- Best for general daily tasks
- Limited to more structured, step-by-step actions

#### Command-Line Interface (CLI)
- Text-based interface where users type commands
- More powerful and efficient for advanced users
- Allows:
  - Automation
  - Running multiple commands
  - Faster system control
- Commonly used in cybersecurity and system administration

### Comparison Summary:
- GUI → easy, visual, slower for complex tasks
- CLI → fast, powerful, requires knowledge

### Importance in Cybersecurity:
- Analysts use both interfaces
- CLI helps with:
  - Logging actions (command history)
  - Incident response
  - System troubleshooting
