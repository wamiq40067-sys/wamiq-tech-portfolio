## Navigate the Linux File System

- Linux organizes files using the Filesystem Hierarchy Standard (FHS).
- The root directory (`/`) is the starting point of the Linux filesystem.
- Understanding absolute and relative file paths is essential for navigation.
- Commands like `pwd`, `ls`, and `cd` are fundamental for moving through directories.
- Special symbols such as `~`, `.`, and `..` simplify navigation.

---

## Manage File Content in Bash

- Linux provides powerful commands for viewing and managing files directly from the terminal.
- Commands such as `cat`, `head`, `tail`, and `less` help inspect file contents efficiently.
- `grep`, pipes (`|`), and `find` are essential for searching and filtering information.
- File and directory management commands include `touch`, `rm`, `mkdir`, `mv`, and `cp`.
- `nano` is a beginner-friendly text editor commonly used in Linux.
- Output redirection operators (`>` and `>>`) allow writing or appending command output to files.

---

## Authenticate and Authorize Users

- Authentication confirms user identity, while authorization controls access permissions.
- Linux permissions are divided into read (`r`), write (`w`), and execute (`x`) permissions.
- Permissions apply to three owner categories: user, group, and others.
- `chmod` modifies permissions, and `chown` changes file ownership.
- `sudo` provides temporary elevated privileges for administrative tasks.
- The principle of least privilege helps reduce security risks by limiting unnecessary access.
- User management commands like `useradd`, `usermod`, and `userdel` are essential for system administration.
