Common Operating Systems

Several operating systems are widely used in the cybersecurity field, including Windows, macOS, Linux, ChromeOS, Android, and iOS. Understanding these systems is important because each has unique security features, architectures, and vulnerabilities.

Windows and macOS

Windows and macOS are two of the most commonly used operating systems for personal and business computers.

Windows, introduced in 1985, is a closed-source operating system. This means its source code is not publicly available.
macOS, first released in 1984, combines both open-source and proprietary components. Some parts, such as the kernel, are open source, while other sections remain private.

Both operating systems are heavily used in enterprise environments and are frequent targets for cyberattacks due to their popularity.

Linux

Linux was first released in 1991 and is fully open source. Anyone can access, modify, and distribute its source code. Because of this openness, developers worldwide contribute to improving Linux distributions.

Linux plays a major role in cybersecurity because many security tools and specialized operating systems are built on it. Certain Linux distributions are specifically designed for penetration testing, digital forensics, and security analysis.

ChromeOS

ChromeOS became available in 2011 and is based on Chromium OS. Chromium OS is entirely open source, while ChromeOS includes both open-source and proprietary elements.

ChromeOS is commonly used in educational institutions because of its lightweight design, cloud integration, and simplified management features.

Android and iOS

Android and iOS are operating systems designed primarily for mobile devices such as smartphones, tablets, and smartwatches.

Android was publicly released in 2008 and is largely open source.
iOS, introduced in 2007, is partially open source with many proprietary components.

Mobile operating systems have become increasingly important in cybersecurity due to the growth of mobile applications, cloud services, and mobile-based attacks.

Operating Systems and Security Risks

All operating systems can contain vulnerabilities, regardless of how secure they are designed to be. One of the most effective ways to reduce security risks is by regularly updating the operating system and its software components.

Legacy Operating Systems

A legacy operating system refers to software that is outdated but still in use. Organizations may continue using older systems because certain applications or hardware are incompatible with newer versions.

This is especially common in industries that rely on specialized equipment with embedded software.

Legacy systems create significant security concerns because vendors may stop providing updates and patches. Without ongoing support, these systems become more exposed to newly discovered threats and attacks.

Additional Vulnerabilities

Even fully updated operating systems can still face security issues. Cybersecurity professionals often monitor official vulnerability databases and vendor security bulletins to stay informed about emerging threats.

Examples include:

Microsoft Security Response Center (MSRC) for Microsoft products
Apple Security Updates for macOS and iOS
Ubuntu CVE reports for Linux vulnerabilities
Google Cloud Security Bulletins for Google-related services

Staying informed about vulnerabilities and maintaining updated systems are essential responsibilities for security analysts and IT professionals.

Booting a Computer

When a computer is powered on, a small chip called either the BIOS or UEFI begins the startup process.

BIOS (Basic Input/Output System) is an older firmware technology that contains startup instructions for the computer.
UEFI (Unified Extensible Firmware Interface) is the modern replacement for BIOS and includes improved functionality and stronger security features.

Both BIOS and UEFI perform similar tasks during startup. Their role is to initialize the system, check hardware components, and prepare the computer to load the operating system. UEFI became more common after 2007 and is now standard in most modern computers.

One important responsibility of BIOS or UEFI is verifying that hardware components are functioning properly. After these checks are completed, the firmware activates the bootloader, which is a program responsible for loading the operating system into memory.

Once the operating system finishes loading, the computer becomes ready for normal use.

How a Computer Completes Tasks

After startup, computers perform tasks through cooperation between four main components:

User
Application
Operating System
Hardware
1. User

The process begins with the user. A user starts an action they want the computer to perform, such as opening a website, editing a document, or downloading a file.

2. Application

Applications are software programs designed to help users complete specific tasks.

Examples include:

A calculator app for mathematical operations
A word processor for creating documents
A web browser for accessing the internet

The application receives the user’s input and forwards the request to the operating system.

3. Operating System

The operating system (OS) acts as a manager between software and hardware. It interprets requests from applications and decides how system resources should be used to complete the task.

The OS communicates with hardware components and coordinates the work required for processing data.

4. Hardware

Hardware components perform the actual processing work.

Examples include:

The CPU handling calculations and instructions
The hard drive or SSD storing and retrieving files
Memory components temporarily holding data during processing

After the hardware completes the requested action, the result is returned to the operating system, then back to the application, where it is displayed to the user.

Understanding the OS Through Analogies
Car Analogy

Using a computer is similar to driving a car. A driver presses the accelerator and the car moves forward, even though the driver may not fully understand the engine’s internal processes.

In the same way, users interact with applications while the operating system quietly handles complex background operations.

Restaurant Analogy

The operating system can also be compared to a restaurant kitchen.

The customer represents the user.
The food order represents the request made through an application.
The kitchen staff represent the operating system and hardware working behind the scenes.

A customer only sees the final meal, not the preparation process. Similarly, computer users only see the completed result, while the OS manages the internal operations required to fulfill the request.

Example: Downloading a File

The process of downloading a file demonstrates how users, applications, the OS, and hardware work together.

Step 1: User Action

The user clicks a download button in a web browser.

Step 2: Application Request

The web browser sends the download request to the operating system.

Step 3: OS Coordination

The operating system processes the request and directs the appropriate hardware components to begin downloading the file.

Step 4: Hardware Processing

The hardware handles network communication and stores the downloaded file on the storage device.

Step 5: Output to User

After the download is complete, the operating system informs the browser application, which then notifies the user that the file has been successfully downloaded.













What is a Virtual Machine?

A virtual machine (VM) is a software-based version of a physical computer. It behaves like a real computer even though it does not exist as separate physical hardware.

Virtual machines are created through a technology called virtualization. Virtualization uses software to simulate physical hardware components such as:

CPU
Storage
Memory (RAM)
Network interfaces

Because of virtualization, a single physical computer can run multiple independent virtual machines at the same time.

How Virtual Machines Work

A physical computer, known as the host machine, shares its hardware resources with several virtual machines.

Each VM functions like an independent computer and usually includes:

Its own operating system
Applications
Virtual hardware resources

For example, if a host computer has 16 GB of RAM, the memory can be divided among multiple virtual machines and the host system itself. Three virtual machines could each receive 4 GB of RAM while the remaining memory is used by the host operating system.

Even though the virtual machines share the same physical hardware, they operate separately from one another.

Benefits of Virtual Machines

Virtual machines are widely used in cybersecurity and IT because they improve both security and efficiency.

Security Advantages

One major advantage of virtualization is isolation. Virtual machines operate in separate environments called sandboxes.

Because VMs are isolated:

Problems inside one VM usually do not affect others
Malware infections can be contained more safely
Security professionals can analyze suspicious files in controlled environments

For example, cybersecurity analysts often run malware inside a VM to study its behavior without risking the main system.

However, virtualization is not completely risk-free. Some advanced malicious software may escape the virtual environment and access the host machine. Because of this, virtual machines should still be used carefully and securely.

Efficiency Advantages

Virtualization also improves efficiency by allowing multiple systems to run on one physical computer.

Benefits include:

Running several operating systems simultaneously
Quickly switching between environments
Testing software without separate hardware
Reducing equipment costs

A useful comparison is a city bus. Instead of every person driving a separate car, many passengers share one vehicle. Similarly, many virtual machines can share one physical computer, reducing hardware usage and resource waste.

Managing Virtual Machines

Virtual machines are controlled using software called a hypervisor.

A hypervisor:

Creates and manages virtual machines
Connects virtual hardware to physical hardware
Allocates system resources such as CPU power, RAM, and storage

One important hypervisor is KVM (Kernel-based Virtual Machine).

KVM:

Is open source
Works with many Linux distributions
Is built directly into the Linux kernel
Allows Linux systems to create and manage virtual machines without requiring additional virtualization software
Other Types of Virtualization

Virtualization is not limited to virtual machines. Other forms include:

Virtual servers: Multiple server environments created from one physical server
Virtual networks: Software-based networks that improve resource usage and flexibility
Storage virtualization: Combining multiple storage devices into a single virtual storage system

These technologies help organizations use hardware resources more effectively while improving scalability and management.

CLI vs GUI

Computers can be interacted with through different types of user interfaces. Two of the most common are the Graphical User Interface (GUI) and the Command-Line Interface (CLI).

A GUI allows users to interact with the computer using graphical elements such as icons, buttons, and menus.
A CLI is a text-based interface where users type commands to communicate with the system.

Both interfaces are important in computing and cybersecurity, and each has its own strengths.

Display Differences

One major difference between a GUI and a CLI is how they appear to users.

Graphical User Interface (GUI)

A GUI includes visual components such as:

Icons
Windows
Menus
Toolbars

Examples include desktop environments where users click icons to open applications or manage files.

GUIs are designed to be visually friendly and easier for beginners to navigate.

Command-Line Interface (CLI)

A CLI contains only text. Users interact with the system by entering commands into a terminal or command prompt.

Instead of clicking buttons, users type instructions directly for the operating system to execute.

The interface often resembles programming code because it displays lines of text-based commands and outputs.

Functional Differences

GUIs and CLIs also differ in how tasks are performed.

GUI Functionality

A GUI generally handles one action at a time through visual interaction. Users complete tasks step-by-step by clicking through menus or windows.

For example:

Opening folders
Dragging files
Clicking application icons

This makes GUIs convenient for everyday tasks and beginner users.

CLI Functionality

A CLI allows users to perform multiple operations quickly using commands. Complex tasks can often be automated or combined into a single command.

This makes CLIs especially powerful for system administration, scripting, and cybersecurity operations.

Advantages of CLI in Cybersecurity

Cybersecurity professionals commonly use both GUIs and CLIs, but the CLI offers several important advantages.

Efficiency

Experienced users can often work faster in a CLI because commands allow direct interaction with the operating system.

For example, creating many files through a GUI may require repeating the same steps multiple times. In a CLI, a single command can complete the task instantly.

CLIs are also useful for:

Automating repetitive tasks
Managing systems remotely
Running scripts
Processing large amounts of data quickly
Command History

One major benefit of a CLI is the ability to maintain a history file that records previously executed commands.

This is particularly valuable in cybersecurity because analysts can:

Review the exact steps they performed
Verify procedures during incident response
Troubleshoot mistakes
Audit system activity

For example, when following an incident response playbook, analysts can check command history to confirm that all required commands were executed correctly.

Command history may also help investigators track suspicious or malicious actions if a system has been compromised.

Choosing Between CLI and GUI

The choice between a GUI and CLI often depends on the task and the user’s experience level.

GUIs are generally easier for beginners and casual users.
CLIs provide greater speed, flexibility, and control for advanced tasks.

In cybersecurity and system administration, learning both interfaces is important because each is useful in different situations.

Linux System Components
User

The user is the individual who interacts with the computer system. Users start, control, and manage tasks on the computer. Linux is known as a multi-user operating system, meaning several users can access and use the same system resources simultaneously.

Applications

Applications are software programs designed to perform specific tasks for users. Some applications, like calculators and calendars, are usually pre-installed on a computer, while others, such as web browsers or email clients, may need to be installed separately.

In Linux, applications are commonly installed using a package manager. A package manager is a utility that helps users install, update, manage, and remove software packages. A package contains software components that work together to create an application.

Shell

The shell acts as the command-line interface between the user and the operating system. It interprets text-based commands entered by the user and communicates them to the kernel for execution.

The shell can be considered a translator between the user and the computer, converting user commands into instructions the system can understand and process.

Filesystem Hierarchy Standard (FHS)

The Filesystem Hierarchy Standard (FHS) defines how data and directories are organized within the Linux operating system. It determines where files, programs, and system data are stored so the OS can locate them efficiently.

A directory, also called a folder, is used to organize files and other directories. The FHS provides a structured layout for storing and managing data across the system.

Kernel

The kernel is the core component of the Linux operating system. It manages system resources such as processes and memory and serves as the bridge between software applications and hardware.

The Linux kernel is responsible for handling major hardware operations, allocating resources, and ensuring efficient communication between applications and the computer’s physical components.

Hardware

Hardware refers to the physical parts of a computer system. Examples include hard drives, processors, keyboards, and monitors. Hardware is generally divided into two categories: peripheral devices and internal hardware.

Peripheral Devices

Peripheral devices are external hardware components connected to the computer system. These devices are not essential for the computer to function and can usually be added or removed easily.

Examples include:

Monitor
Keyboard
Mouse
Printer
Internal Hardware

Internal hardware consists of the essential components required for a computer to operate. These parts are connected to the motherboard, the main circuit board of the computer.

Key internal hardware components include:

Central Processing Unit (CPU):
The CPU is the primary processor of the computer. It executes instructions from programs and performs general computing tasks.
Random Access Memory (RAM):
RAM provides temporary or short-term memory for the computer. It stores data currently being used by programs and processes. Data in RAM is lost when the computer is turned off.
Hard Drive:
The hard drive is used for long-term storage of files, programs, and operating system data. Unlike RAM, data stored on a hard drive remains available even after the computer is powered off. Some systems may contain multiple hard drives.

Popular Linux Distributions Used in Cybersecurity
Kali Linux

Kali Linux is a widely recognized open-source Linux distribution commonly used by cybersecurity professionals. It is specifically designed for security-related tasks and comes with a large collection of pre-installed tools for activities such as penetration testing and digital forensics.

Penetration testing involves simulating cyberattacks to discover weaknesses in systems, applications, or networks before real attackers can exploit them. Digital forensics focuses on gathering and examining digital evidence after a security incident to understand what occurred.

Although Kali Linux is very popular in cybersecurity, it is not the only Linux distribution used in the field.

Ubuntu

Ubuntu is a beginner-friendly and widely adopted Linux distribution used across many industries, including cybersecurity. It offers both a graphical user interface (GUI) and a command-line interface (CLI), making it suitable for both new and experienced users.

Ubuntu is based on Debian and includes several essential applications by default. Additional software, including security tools, can be installed through its package management system. Due to its large user community, Ubuntu has extensive online documentation and support resources.

Ubuntu is also commonly used in cloud computing environments, which makes it important for cybersecurity professionals working with cloud infrastructure and servers.

Parrot OS

Parrot OS is another open-source Linux distribution designed for cybersecurity and privacy-focused tasks. Similar to Kali Linux, it includes many built-in tools for penetration testing, ethical hacking, and digital investigations.

Parrot OS is known for being lightweight and user-friendly. It supports both GUI and CLI environments, allowing users to choose how they prefer to interact with the system.

Red Hat Enterprise Linux

Red Hat Enterprise Linux, often called RHEL, is a commercial Linux distribution developed for enterprise and business environments. Unlike many open-source distributions, it requires a paid subscription.

RHEL is valued for its stability, security, and professional support services. Organizations often use it in corporate environments where reliability and dedicated technical assistance are important.

AlmaLinux

AlmaLinux is a free, community-supported Linux distribution created to serve as an alternative to CentOS after CentOS 8 reached end-of-life.

AlmaLinux is designed to maintain compatibility with systems previously running CentOS, allowing organizations to continue using their existing applications and configurations with minimal changes. It is commonly used in servers and enterprise environments that require long-term stability.









Introduction to Package Managers
What is a Package?

A package is a collection of software files used to install and run applications on a Linux system. Some packages work together with other packages to create a complete application, while others may function independently as standalone software.

Packages usually include important components such as program files, configuration files, and dependencies. Dependencies are additional libraries or supporting files required for the application to work correctly.

What is a Package Manager?

A package manager is a utility that helps users install, update, manage, and remove software packages from a system. It simplifies software management by automatically handling dependencies and ensuring that required files are installed properly.

Using updated package versions is important because newer releases often contain security patches, performance improvements, and bug fixes that help protect the system.

Types of Package Managers

Different Linux distributions use different package management systems. Many Linux distributions are based on parent distributions, which determines the package manager they support.

For example:

Kali Linux, Ubuntu, and Parrot OS are based on Debian.
CentOS and other similar distributions are derived from Red Hat.

Because of this relationship, Debian-based systems commonly use package managers such as dpkg, while Red Hat-based systems use RPM.

Debian Package Manager (dpkg)

dpkg is a package manager used in Debian-based Linux distributions. It works with software packages that use the .deb file format.

Example format:

Package_Version-Release_Architecture.deb
Red Hat Package Manager (RPM)

RPM is used in Red Hat-based Linux distributions. It manages packages with the .rpm file extension.

Example format:

Package-Version-Release_Architecture.rpm
Package Management Tools

In addition to package managers, Linux also provides package management tools that make software management easier through the command-line interface.

These tools simplify tasks such as:

Installing software
Updating packages
Searching repositories
Removing applications
Advanced Package Tool (APT)

APT is a command-line package management tool used on Debian-based distributions. It works together with dpkg to install, update, search, and manage packages efficiently.

APT is commonly used on systems like:

Ubuntu
Kali Linux
Parrot OS
Yellowdog Updater Modified (YUM)

YUM is a package management tool designed for Red Hat-based Linux distributions. It helps users manage .rpm packages through the command line.

YUM can:

Install packages
Update software
Search repositories
Resolve dependencies automatically

It is commonly used in Red Hat-related distributions and enterprise environments.

Communicating Through a Shell

A shell is a command-line interface that allows users to interact with the operating system. It works as a bridge between the user and the computer by interpreting commands and sending them to the system for execution.

When a user enters a command, the shell processes the instruction, communicates with the kernel, and then displays the result back to the user. This makes the shell an essential component for managing and controlling Linux systems.

Common Types of Linux Shells

Linux provides several types of shells, each with its own features and functionality. Some commonly used shells include:

Bourne-Again Shell (bash)
C Shell (csh)
Korn Shell (ksh)
Enhanced C Shell (tcsh)
Z Shell (zsh)

Although these shells support many standard Linux commands, they differ in syntax, customization options, and advanced features.

For example:

bash and ksh typically use the dollar sign ($) as the command prompt.
zsh often uses the percent sign (%) as the prompt symbol.
Bash Shell

Bash, short for Bourne-Again Shell, is the default shell for most Linux distributions. It is widely preferred because of its simplicity, flexibility, and user-friendly design.

Bash supports:

Basic Linux commands
Shell scripting
Automation tasks
System administration activities

It is also one of the most commonly used shells in cybersecurity and system administration, making it an important tool for learning Linux and security operations.
