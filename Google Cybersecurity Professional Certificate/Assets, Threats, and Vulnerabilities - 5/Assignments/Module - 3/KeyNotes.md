# Flaws in the System

## What are System Flaws?
- System flaws are weaknesses or mistakes in hardware, software, networks, or processes that can create security risks.
- Attackers exploit these weaknesses to gain unauthorized access, steal data, or disrupt operations.

## Common Causes of System Flaws
- Poor system design
- Misconfigured security settings
- Outdated software
- Weak passwords
- Unpatched vulnerabilities
- Human error
- Insecure third-party components

## Types of Vulnerabilities
- Software vulnerabilities
- Hardware vulnerabilities
- Network vulnerabilities
- Human-related vulnerabilities
- Cloud and application vulnerabilities

## Attack Surface
- The attack surface includes every possible entry point attackers can exploit.
- Examples:
  - Servers
  - Devices
  - Applications
  - User accounts
  - Wireless networks

## Security Hardening
- Security hardening is the process of reducing vulnerabilities and minimizing the attack surface.
- Examples:
  - Disabling unused services
  - Updating systems regularly
  - Enforcing strong password policies
  - Limiting user permissions

## Defense in Depth
- A layered security approach used to reduce risk.
- Uses multiple controls instead of relying on a single defense mechanism.

# Identify System Vulnerabilities

## Vulnerability Assessment
- An internal review process used to identify weaknesses in systems and networks.
- Helps organizations discover risks before attackers exploit them.

## Vulnerability Management
- Continuous process of:
  - Identifying vulnerabilities
  - Assessing risks
  - Prioritizing issues
  - Applying fixes and patches

## Vulnerability Scanners
- Automated tools that compare systems against known vulnerabilities and exposures.
- Commonly used to scan:
  - Networks
  - Servers
  - Applications
  - Endpoints

## Types of Scans

### External Scans
- Analyze systems exposed to the internet.
- Focus on perimeter defenses.

### Internal Scans
- Examine internal systems and applications.

### Authenticated Scans
- Use valid login credentials to identify deeper vulnerabilities.

### Unauthenticated Scans
- Simulate attacks from external users without access.

### Limited Scans
- Focus on specific devices or systems.

### Comprehensive Scans
- Analyze the entire network and connected systems.

## Common Vulnerabilities and Exposures (CVE®)
- Public database of known vulnerabilities and security flaws.

## Common Vulnerability Scoring System (CVSS)
- Framework used to measure vulnerability severity.

## Zero-Day Vulnerabilities
- Previously unknown vulnerabilities that attackers may exploit before a fix is available.

## Patch Management
- Process of updating systems and software to fix vulnerabilities.
- Important for preventing exploitation of known flaws.

## Penetration Testing
- Simulated cyberattacks used to test security defenses.
- Helps determine the impact of vulnerabilities.

# Cyber Attacker Mindset

## What is an Attacker Mindset?
- Thinking like a threat actor to understand how systems could be attacked.
- Helps security teams predict attack methods and improve defenses.

## Purpose of Applying an Attacker Mindset
- Identify weak points
- Test security controls
- Anticipate attacker behavior
- Improve incident response

## Threat Actors
- Any person or group that poses a cybersecurity risk.

### Types of Threat Actors
- Competitors
- Criminal syndicates
- Insider threats
- State-sponsored actors
- Hacktivists
- Shadow IT users

## Types of Hackers

### Unauthorized (Malicious) Hackers
- Break into systems illegally for personal gain or disruption.

### Authorized (Ethical) Hackers
- Test systems legally to improve security.

### Semi-Authorized Hackers
- May violate rules for political or social causes.

## Advanced Persistent Threats (APTs)
- Long-term attacks where threat actors remain hidden inside systems.
- Often associated with nation-state or highly organized attackers.

## Attack Vectors
- Pathways attackers use to gain access to systems.

### Common Attack Vectors
- Email phishing
- Social engineering
- USB devices
- Wireless networks
- Cloud services
- Supply chain attacks
- Weak passwords

## Red Team and Blue Team Exercises

### Red Team
- Simulates attacker behavior to test defenses.

### Blue Team
- Focuses on defense and incident response.

### Purple Team
- Combines red and blue team collaboration to improve security.

## OSINT (Open-Source Intelligence)
- Gathering intelligence from publicly available information.
- Used to identify threats and vulnerabilities.

## Importance of Security Awareness
- Employees are common targets of cyberattacks.
- Training helps reduce risks from phishing, social engineering, and unsafe behavior.
