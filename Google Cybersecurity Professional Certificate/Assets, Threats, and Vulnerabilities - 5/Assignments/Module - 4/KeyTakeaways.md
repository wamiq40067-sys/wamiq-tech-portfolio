# Key Takeaways: Cybersecurity Topics

## Social Engineering

* Exploits human behavior rather than technical system weaknesses.
* Relies on deception tactics like trust, urgency, fear, or curiosity.
* Common forms include phishing, smishing, vishing, baiting, and tailgating.
* Can bypass technical defenses by targeting users directly.
* Strong awareness and verification habits are critical defenses.

---

## Malware

* Malicious software designed to damage, disrupt, or gain unauthorized access to systems.
* Can spread through emails, downloads, networks, or compromised websites.
* Includes viruses, worms, trojans, ransomware, spyware, adware, and rootkits.
* Some malware operates stealthily (fileless malware, botnets, cryptojacking).
* Prevention relies on updates, antivirus tools, safe browsing, and backups.

---

## Web-Based Exploits

* Target vulnerabilities in web applications and websites.
* Common attacks include SQL injection and Cross-Site Scripting (XSS).
* Exploit poor input handling and insecure coding practices.
* Can lead to data theft, session hijacking, or system compromise.
* Prevention includes input validation, sanitization, prepared statements, and secure coding.

---

## Threat Modeling

* A structured process for identifying assets, threats, and vulnerabilities.
* Helps assess risks and improve system security before attacks occur.
* Common frameworks include STRIDE, PASTA, Trike, and VAST.
* Involves analyzing system design, data flow, and potential attack paths.
* Most effective when integrated throughout the software development lifecycle (SDLC).
