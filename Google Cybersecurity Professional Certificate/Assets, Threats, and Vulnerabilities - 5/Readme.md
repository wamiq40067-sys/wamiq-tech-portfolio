# Assets, Threats, and Vulnerabilities

This file contains my learning notes, key concepts, and understanding of foundational cybersecurity concepts including asset management, threat analysis, vulnerability assessment, risk management, data security, and security governance as part of the Google Cybersecurity Professional Certificate.

---

## Topic Overview

This course introduces the core elements that define cybersecurity risk and defense strategies: **assets, threats, vulnerabilities, and risk management principles**. It explains how organizations identify valuable resources, classify and protect them, and assess potential risks that could impact confidentiality, integrity, and availability.

The course also covers how data is managed in different states, how security policies and governance structures support organizations, and how frameworks like the **NIST Cybersecurity Framework (CSF)** help standardize and guide cybersecurity practices across industries.

## Key Topics Covered

### Introduction to Assets
- Definition of an asset as anything valuable to an organization
- Types of assets:
  - Digital assets (databases, customer data)
  - Physical assets (hardware, devices, infrastructure)
  - Information systems (applications, networks)
  - Intangible assets (reputation, intellectual property)

---

### Asset Management and Inventory
- Importance of knowing and tracking all organizational assets
- Asset inventory as a structured catalog of resources
- Asset management as an ongoing process of tracking and risk evaluation
- Importance of knowing:
  - What assets exist
  - Where they are located
  - Who owns them
  - How critical they are

---

### Asset Classification
- Purpose of categorizing assets based on sensitivity and value
- Common classification levels:
  - Restricted
  - Confidential
  - Internal-only
  - Public
- Helps organizations prioritize protection efforts and reduce risk exposure

---

### Risk, Threats, and Vulnerabilities
- Risk defined as likelihood × impact
- Threats as events or conditions that can cause harm
- Types of threats:
  - Intentional (cyberattacks, malicious actions)
  - Unintentional (human error, mistakes)
- Vulnerabilities as weaknesses that can be exploited
- Types of vulnerabilities:
  - Technical (software misconfigurations, system flaws)
  - Human (carelessness, lack of awareness)

---

### Data Security and Information Protection
- Understanding data as processed and stored digital information
- Data states:
  - Data at rest (stored data)
  - Data in transit (moving across networks)
  - Data in use (actively accessed data)
- Importance of protecting data in all states
- Introduction to Information Security (InfoSec)

---

### Policies, Standards, and Governance
- Policies as formal rules for reducing organizational risk
- Procedures as step-by-step instructions for security tasks
- Standards as guidelines used to build policies
- Regulations as legally enforced rules from authorities
- Role of governance in managing security decisions

---

### NIST Cybersecurity Framework (CSF)
- Overview of a widely used cybersecurity framework
- Core functions:
  - Identify
  - Protect
  - Detect
  - Respond
  - Recover
  - Govern
- Framework components:
  - Core (security outcomes)
  - Tiers (maturity levels)
  - Profiles (custom security alignment)
- Helps organizations structure and improve cybersecurity programs
