# Key Notes: security hardening

## OS Hardening

Operating System (OS) hardening refers to the process of securing an operating system by reducing vulnerabilities and minimizing attack surfaces.

### Key Practices:
- Remove unnecessary applications and services
- Apply regular **patch updates** to fix security vulnerabilities
- Enforce strong **password policies**
- Enable **multi-factor authentication (MFA)**
- Restrict user privileges using the principle of least privilege
- Disable unused ports and services
- Configure secure system settings and permissions
- Use antivirus and endpoint protection tools

### Benefits:
- Reduces risk of malware infection
- Limits unauthorized access
- Improves overall system stability and security

---

## Network Hardening

Network hardening focuses on securing network infrastructure to prevent unauthorized access, attacks, and data breaches.

### Key Practices:
- Configure and maintain **firewalls** to filter traffic
- Use **IDS/IPS systems** to detect and prevent attacks
- Implement secure network protocols (HTTPS, SSH, etc.)
- Segment networks using VLANs or subnets
- Regularly monitor traffic using log analysis tools
- Block suspicious IP addresses and malicious traffic patterns
- Protect against DDoS attacks using rate limiting and filtering

### Benefits:
- Reduces risk of external and internal attacks
- Improves visibility of network activity
- Enhances detection and response capabilities

---

## Cloud Hardening

Cloud hardening refers to securing cloud environments by properly configuring services, access controls, and data protection mechanisms.

### Key Practices:
- Proper configuration of **Identity and Access Management (IAM)**
- Enforce least privilege access for all cloud users
- Use encryption for data at rest and in transit
- Secure cloud storage and database configurations
- Implement logging and monitoring tools (SIEM integration)
- Use secure key management systems (TPM, CloudHSM)
- Apply baseline configurations for cloud resources
- Regular security audits and compliance checks

### Benefits:
- Prevents misconfiguration-related breaches
- Secures sensitive cloud data
- Improves visibility and compliance in cloud environments
- Reduces risks from shared infrastructure models
