Connect and Protect: Networks and Network Security


Network - A group of connected devices.

Local Area Network (LAN) - A network that spans a small area like an office building, a school, or a home.

Wide Area Network (WAN) - A network that spans a large geographic area like a city, state, or country.

Hub - A network device that broadcasts information to every device on the network.

Switch - A device that makes connections between specific devices on a network by sending and receiving data between them.

Router - A network device that connects multiple networks together.

Modem - A device that connects your router to the internet and brings internet access to the LAN.

Virtualization tools - Pieces of software that perform network operations.





Network Devices
Network devices help maintain communication and services across a network. These devices connect through wired or wireless connections and exchange information using data packets. Each data packet contains details about the sender and the destination, allowing information to travel correctly between devices.
A network is the complete infrastructure that enables devices to communicate with one another. Specialized devices such as routers and switches manage the flow of data across the network, while devices like computers, smartphones, and tablets connect through them to access network resources.
Basic Network Structure
In a typical home or office setup:


A modem connects the network to the internet through an Internet Service Provider (ISP).


A firewall monitors and filters incoming and outgoing traffic to improve security.


A router distributes internet connectivity to devices within the network.


Devices such as computers, phones, tablets, and printers communicate through the router.


A server may store files or provide services accessible to all connected devices.


A switch can be added to expand the number of available wired connections.


Multiple routers may also be connected for load balancing to improve network performance.



Devices and Desktop Computers
Common devices connected to networks include desktop computers, laptops, smartphones, and tablets. Each device has:


A unique MAC address for hardware identification


An IP address for network communication


A network interface card (NIC) that sends and receives data packets


These devices can connect using either wired Ethernet connections or wireless Wi-Fi connections.

Firewalls
A firewall is a security device that monitors and controls network traffic entering or leaving a network. It acts as an important first layer of defense by enforcing security rules configured by an organization.
Firewalls are usually placed between:


The trusted internal network


External and untrusted networks such as the internet


They help block malicious traffic and restrict unauthorized access, although they are only one component of overall cybersecurity protection.

Servers
Servers provide resources, information, and services to devices on a network. Devices that request services from a server are called clients.
This setup is known as the client-server model, where:


Clients send requests


Servers process the requests


Servers return the requested information or service


Common types of servers include:


DNS servers for domain name lookups


File servers for storing and retrieving files


Mail servers for managing organizational email systems



Hubs and Switches
Hubs
A hub is a basic networking device that connects multiple devices together. It broadcasts all incoming data to every connected port.
Because all traffic is shared with every device, hubs are vulnerable to eavesdropping and are considered less secure. They are rarely used in modern enterprise networks and are mostly limited to small or simple setups.
Switches
Switches are more advanced and commonly used in modern networks. A switch:


Sends data only to the intended device


Uses a MAC address table to identify connected devices


Improves network efficiency and security


Switches operate at the Data Link Layer of the TCP/IP model.

Routers
Routers connect different networks and direct data traffic using IP addresses. They determine the best path for data packets to reach their destination network.
Routers:


Read destination IP information from packet headers


Forward packets between networks


Allow communication across multiple networks


Often include firewall capabilities for added security


Routers operate at the Network Layer of the TCP/IP model.

Modems and Wireless Access Points
Modems
A modem connects a home or office network to an ISP. It converts internet signals into a format compatible with local devices.
ISPs may provide internet through:


Telephone lines


Coaxial cables


Fiber optic cables


The modem usually connects to a router, which then distributes connectivity throughout the network.
Large enterprise networks often use more advanced broadband technologies instead of traditional modems.
Wireless Access Points (WAPs)
A wireless access point creates a wireless network using radio waves. Devices connect through Wi-Fi using wireless adapters.
Wireless access points:


Send and receive wireless signals


Allow devices to connect without cables


Forward traffic to routers and switches for further transmission


Wi-Fi refers to the wireless communication standards used by these devices.

Importance of Network Diagrams
Network diagrams visually represent the structure of a network. They show:


Devices connected to the network


Relationships between devices


Paths used for communication


Administrators and security analysts use these diagrams to:


Understand network architecture


Identify potential vulnerabilities


Plan security strategies


Improve network management and troubleshooting


These diagrams use symbols and connection lines to simplify complex network layouts and make analysis easier.

Cloud network - A collection of servers or computers that store resources and data in remote data centers that can be accessed via the internet.

Cloud service providers offer:
On-demand storage

Cloud Computing and Network Processes

Traditional networks are commonly known as on-premise networks, where all networking equipment, servers, and storage devices are physically located within a company’s own facilities, such as offices or data centers. In this setup, the organization is responsible for purchasing, maintaining, upgrading, and securing all hardware and software components.

Cloud computing works differently. Instead of relying entirely on hardware stored at a company’s physical location, businesses can use computing resources hosted remotely over the internet. These resources may include servers, applications, databases, networking services, and storage systems that are managed by external providers.

A company that offers these online computing resources is known as a Cloud Service Provider (CSP). CSPs operate massive data centers around the world that contain large numbers of servers and networking equipment. Through these data centers, they provide services such as computing power, storage, databases, and networking solutions to organizations on a subscription or usage-based model.

Organizations access cloud services through:

Web-based management consoles
APIs (Application Programming Interfaces)
Remote management tools

This allows companies to use advanced computing infrastructure without having to build and maintain it themselves.

Main Types of Cloud Services

Cloud service providers generally offer three primary service models: SaaS, IaaS, and PaaS.

Software as a Service (SaaS)

Software as a Service allows organizations to use software applications that are fully managed by the cloud provider. The software runs on the provider’s infrastructure, and users access it remotely through a web browser or internet connection.

With SaaS:

Companies do not need to install or maintain the software locally
Updates and maintenance are handled by the provider
Users can access applications from different devices and locations

Examples include cloud-based email platforms, office productivity tools, and collaboration applications.

Infrastructure as a Service (IaaS)

Infrastructure as a Service provides virtualized computing resources through the cloud. Instead of buying physical servers or storage devices, organizations can rent these resources from a CSP.

IaaS services may include:

Virtual machines
Cloud storage
Networking components
Containers
Load balancers

These resources are managed remotely through APIs or web dashboards. Organizations can run their existing applications on cloud infrastructure with little or no major redesign. Additionally, businesses can take advantage of cloud-specific features such as enhanced security, availability, and performance optimization.

Platform as a Service (PaaS)

Platform as a Service provides a development environment where programmers can build, test, and deploy custom applications without managing the underlying infrastructure.

PaaS platforms usually offer:

Development frameworks
Databases
Testing tools
Deployment services
Application hosting environments

This allows developers to focus mainly on writing and improving applications while the cloud provider manages the servers, operating systems, and networking infrastructure.

Hybrid and Multi-Cloud Environments

Many organizations combine cloud services with their existing on-premise systems. This setup is called a hybrid cloud environment.

In a hybrid cloud model:

Some resources remain within the company’s physical infrastructure
Other services operate through cloud providers

Organizations often use hybrid environments to balance flexibility, cost savings, and control over sensitive systems or data.

When a company uses services from multiple cloud providers simultaneously, it is known as a multi-cloud environment. Businesses may choose this strategy to improve redundancy, reduce dependency on a single provider, or select specialized services from different vendors.

Today, hybrid cloud adoption is extremely common because it allows organizations to modernize their infrastructure while still maintaining some direct control over critical operations.

Software-Defined Networking (SDN)

Cloud providers also offer virtual networking technologies similar to physical networking hardware found in traditional environments.

Software-Defined Networking (SDN) refers to networking systems that use software-based controls instead of relying only on dedicated hardware devices. In SDN environments, virtual versions of networking components are created and managed through software.

Examples of virtualized network devices include:

Virtual switches
Virtual routers
Virtual firewalls
Virtual load balancers

These services operate within the cloud provider’s infrastructure and are controlled through software interfaces rather than manual hardware configuration.

Modern physical networking devices also support software-defined features, allowing network management and traffic routing to be handled programmatically.

In cloud environments, SDN tools are hosted within the CSP’s data centers, enabling organizations to quickly configure and manage networks remotely.

Advantages of Cloud Computing and SDNs

Cloud computing and software-defined networking provide several major benefits to organizations. The most significant advantages include reliability, cost efficiency, and scalability.

Reliability

Cloud services are designed to provide consistent availability and stable performance. Reliability in cloud environments depends on factors such as:

Service uptime
Secure connections
Data redundancy
Continuous operation of systems

Because cloud providers operate large-scale infrastructures with backup systems and global data centers, users can often access applications and services with minimal downtime or interruptions.

This reliability supports both employees and customers by ensuring continuous access to important resources and services.

Reduced Costs

Traditional network infrastructure often requires substantial investment in hardware, software licenses, maintenance, upgrades, and technical staff.

Cloud computing reduces these expenses because organizations can rent resources instead of purchasing and managing everything themselves.

Cloud providers benefit from economies of scale since they operate enormous data centers capable of serving many customers simultaneously. As a result:

Businesses avoid major upfront hardware costs
Maintenance and updates are handled by the CSP
Organizations only pay for the resources they use

This makes advanced technology solutions more affordable for companies of different sizes.

Scalability

Scalability refers to the ability of systems to grow or shrink based on changing business demands.

In traditional environments, scaling often requires purchasing additional equipment, which can be expensive and time-consuming. If demand later decreases, the organization may be left with unused infrastructure and unnecessary expenses.

Cloud services solve this issue through an elastic usage model. Companies can quickly increase or decrease resources depending on current needs.

For example, businesses can:

Expand storage capacity
Add virtual servers
Increase network bandwidth
Deploy additional security tools

These changes can often be completed within minutes using APIs or web management consoles instead of physically installing new hardware.

Cloud scalability also improves security response capabilities. Organizations can rapidly deploy protective technologies such as:

Web Application Firewalls (WAFs)
Intrusion Detection and Prevention Systems (IDS/IPS)
Layer 3 and Layer 4 firewalls

This flexibility allows businesses to respond faster to threats while maintaining strong network performance and operational efficiency.

The TCP/IP Model

The TCP/IP model is a networking framework used to explain how data travels across networks and the internet. It helps network engineers, administrators, and cybersecurity professionals understand how communication occurs between devices and how different networking processes interact with one another.

This model is especially useful when identifying network problems or investigating security incidents because it organizes communication into separate layers. By determining which layer is affected, professionals can better locate failures, attacks, or misconfigurations within a network.

The TCP/IP model consists of four layers:

Network Access Layer
Internet Layer
Transport Layer
Application Layer

Each layer performs specific tasks that contribute to the successful transmission of data from one device to another.

Network Access Layer

The Network Access Layer, also called the Data Link Layer, is responsible for handling the physical transmission of data across a local network. This layer manages how data packets are created and transferred between devices connected to the same network.

It includes the hardware and physical components used for communication, such as:

Network cables
Switches
Hubs
Modems
Network interface cards (NICs)
Wireless connections

One important protocol operating at this layer is the Address Resolution Protocol (ARP).

Address Resolution Protocol (ARP)

ARP is used to associate an IP address with a device’s physical MAC address on a local network. Since devices on the same network communicate using MAC addresses, ARP helps identify which MAC address belongs to a specific IP address.

This process allows devices to locate and communicate with each other within the same physical network environment.

Internet Layer

The Internet Layer, sometimes referred to as the Network Layer, is responsible for routing data packets between networks and ensuring that packets reach the correct destination.

This layer adds IP addressing information to packets so that devices know where the data originated and where it should be delivered.

The Internet Layer also determines the protocols used to transport packets across interconnected networks.

Common Protocols at the Internet Layer
Internet Protocol (IP)

The Internet Protocol is responsible for sending packets to the correct destination network. IP assigns addressing information that identifies both the sender and the receiver.

IP works together with transport protocols such as TCP and UDP to complete communication between devices across networks.

In TCP/IP communication:

IP handles addressing and routing
TCP or UDP handles delivery to applications and services

TCP also helps retransmit lost or corrupted data to improve reliability.

Internet Control Message Protocol (ICMP)

ICMP is used to share network status information and report communication errors.

It helps network administrators troubleshoot connectivity problems by providing information about:

Lost or dropped packets
Routing issues
Connectivity failures
Redirected traffic

ICMP is commonly used by network diagnostic tools such as ping.

Transport Layer

The Transport Layer manages end-to-end communication between systems and controls how data is transmitted across the network.

This layer is responsible for:

Data delivery
Traffic flow control
Error handling
Communication reliability

The two primary protocols used at this layer are TCP and UDP.

Transmission Control Protocol (TCP)

Transmission Control Protocol (TCP) is a connection-oriented communication protocol. Before transmitting data, TCP establishes a reliable connection between devices.

TCP ensures:

Data arrives successfully
Packets are received in the correct order
Missing data is retransmitted
Communication errors are detected

TCP also uses port numbers to identify the destination application or service on a device.

Because of its reliability, TCP is commonly used for:

Web browsing
Email services
File transfers
Secure communications
User Datagram Protocol (UDP)

User Datagram Protocol (UDP) is a connectionless protocol that sends data without first establishing a formal connection between devices.

Unlike TCP, UDP:

Does not guarantee delivery
Does not retransmit lost packets
Performs minimal error checking

This makes UDP faster and more efficient for applications where speed is more important than reliability.

UDP is often used for:

Video streaming
Online gaming
Voice calls
Live broadcasts

These applications prioritize real-time performance over perfect accuracy.

Application Layer

The Application Layer is the top layer of the TCP/IP model and is responsible for enabling users and applications to interact with network services.

This layer handles:

Network requests
Responses between applications
User-facing internet services
Data formatting and communication rules

Protocols at this layer determine how applications exchange information across networks.

Common Application Layer Protocols
Hypertext Transfer Protocol (HTTP)

HTTP is used for transferring web pages and website content across the internet.

Simple Mail Transfer Protocol (SMTP)

SMTP is used for sending and transferring email messages between mail servers.

Secure Shell (SSH)

SSH provides secure remote access to systems through encrypted communication.

File Transfer Protocol (FTP)

FTP allows files to be uploaded, downloaded, and transferred between systems.

Domain Name System (DNS)

DNS translates domain names, such as website names, into IP addresses that computers can understand and use for communication.

TCP/IP Model vs OSI Model

The TCP/IP model and the OSI model are both frameworks used to explain how networking and communication operate.

The OSI model divides networking functions into seven layers, while the TCP/IP model simplifies these functions into four layers.

Although structured differently, both models:

Organize network communication into layers
Define networking standards
Help professionals troubleshoot network issues
Improve understanding of data transmission processes

The TCP/IP model combines several OSI layers together, making it a more practical and widely used model in real-world networking environments.

Overall, the TCP/IP model serves as a simplified representation of how communication occurs across modern computer networks and the internet.










TCP/IP and OSI Networking Models

Two of the most important communication protocols used in networking are the Transmission Control Protocol (TCP) and the Internet Protocol (IP).

TCP establishes and manages reliable connections between devices.
IP handles addressing and routing so that data packets can travel across networks and reach the correct destination.

These protocols operate within specific layers of the TCP/IP networking model.

The TCP/IP model is a simplified networking framework made up of four layers, while the OSI (Open Systems Interconnection) model provides a more detailed structure consisting of seven layers. Both models are used to explain how data moves through a network and how communication occurs between devices.

Understanding both models is important for network administrators and cybersecurity professionals because they help identify where communication failures, performance issues, or security threats occur.

TCP/IP Model vs OSI Model

The TCP/IP model organizes network communication into four layers:

Application Layer
Transport Layer
Internet Layer
Network Access Layer

The OSI model expands these networking processes into seven layers:

Application Layer
Presentation Layer
Session Layer
Transport Layer
Network Layer
Data Link Layer
Physical Layer

The TCP/IP model combines several OSI layers into broader categories, making it more streamlined and commonly used in practical networking environments. The OSI model, however, provides a more detailed explanation of networking operations and is widely used for learning, troubleshooting, and security analysis.

Security professionals often use both models to determine where attacks, communication failures, or protocol issues occur during network operations.

Layer 7: Application Layer

The Application Layer is the layer closest to the end user. It includes the protocols and services that applications use to communicate over the internet.

This layer is responsible for enabling user interaction with network services through software applications such as:

Web browsers
Email clients
Messaging applications
File-sharing tools

For example:

Web browsers use HTTP or HTTPS to exchange website data with web servers.
Email systems use SMTP to send and receive messages.
DNS is used to convert domain names into IP addresses so devices can locate web servers.

The defining feature of this layer is that it directly supports communication between users and network applications.

Layer 6: Presentation Layer

The Presentation Layer focuses on data formatting, translation, and encryption. Its job is to ensure that data sent from one system can be correctly interpreted by another system, even if they use different formats.

Functions performed at this layer include:

Data conversion
Compression
Encryption and decryption
Character encoding

One common example is SSL/TLS encryption, which secures communication between web browsers and web servers on HTTPS websites.

The Presentation Layer helps standardize how information is represented across different systems and applications.

Layer 5: Session Layer

The Session Layer manages the establishment, maintenance, and termination of communication sessions between devices.

A session begins when two systems create a connection for communication and ends when the data transfer is complete.

Responsibilities of this layer include:

Session creation and termination
Authentication processes
Session synchronization
Reconnection management
Checkpoint creation during transfers

Checkpoints are especially useful during interruptions because they allow communication to resume from the last successful point instead of restarting the entire transfer.

The Session Layer acts as a coordinator between higher-level applications and lower networking layers.

Layer 4: Transport Layer

The Transport Layer handles the reliable delivery of data between systems.

Its main responsibilities include:

Managing data transfer speed
Controlling traffic flow
Segmenting large data into smaller pieces
Reassembling data at the destination
Detecting transmission errors

Large transmissions are divided into smaller segments so they can travel more efficiently across the network. Once received, the segments are rebuilt into the original data.

The two primary protocols operating at this layer are:

Transmission Control Protocol (TCP)

TCP establishes a reliable connection before sending data. It ensures that:

Data arrives completely
Packets remain in the correct order
Lost data is retransmitted

TCP is commonly used for applications requiring reliable communication, such as web browsing, email, and file transfers.

User Datagram Protocol (UDP)

UDP is a connectionless protocol that sends data without creating a dedicated connection first.

UDP prioritizes speed over reliability and is commonly used in:

Video streaming
Online gaming
Voice and video calls
Live broadcasts

Because it performs less error checking, UDP enables faster communication for real-time applications.

Layer 3: Network Layer

The Network Layer is responsible for moving data packets between networks.

This layer:

Receives frames from the Data Link Layer
Determines packet destinations using IP addresses
Routes packets across networks
Helps routers forward traffic correctly

Data packets contain addressing information that tells routers where the data should be delivered. Routers then guide the packets from the sender’s network to the destination network.

This layer plays a major role in internet communication and routing.

Layer 2: Data Link Layer

The Data Link Layer controls communication within the same local network.

It organizes how devices send and receive data frames across physical network connections.

Devices and technologies associated with this layer include:

Switches
Network Interface Cards (NICs)
MAC addresses

The Data Link Layer helps ensure that data moves correctly between devices connected to the same network infrastructure.

Some protocols operating at this layer include:

Network Control Protocol (NCP)
High-Level Data Link Control (HDLC)
Synchronous Data Link Control (SDLC)

This layer focuses on local delivery rather than routing data across multiple networks.

Layer 1: Physical Layer

The Physical Layer represents the actual hardware and physical connections used for network communication.

This includes:

Ethernet cables
Fiber optic cables
Wireless signals
Hubs
Modems
Connectors
Electrical transmissions

At this layer, data is converted into binary signals consisting of 0s and 1s so it can travel through cables, radio waves, or other transmission media.

The receiving device then interprets these signals and passes the data upward through the higher layers of the OSI model for processing.

The Physical Layer forms the foundation of all network communication because it handles the actual transmission of data between devices.

Operations at the network layer
Functions at the network layer organize the addressing and delivery of data packets across the network from the host device to the destination device. This includes directing the packets from one router to another router across the internet, till it reaches the internet protocol (IP) address of the destination network. The destination IP address is contained within the header of each data packet. This address will be stored for future routing purposes in  routing tables along the packet’s path to its destination.

All data packets include an IP address. A data packet is also referred to as an IP packet for TCP connections or a datagram for UDP connections. A router uses the IP address to route packets from network to network based on information contained in the IP header of a data packet. Header information communicates more than just the address of the destination. It also includes information such as the source IP address, the size of the packet, and which protocol will be used for the data portion of the packet. 

Format of an IPv4 packet
An IP packet divided into two parts: a section on the left marked “header,” and section on the right marked “data”
Next, you can review the format of an IP version 4 (IPv4) packet and review a detailed graphic of the packet header. An IPv4 packet is made up of two sections, the header and the data:

An IPv4 header format is determined by the IPv4 protocol and includes the IP routing information that devices use to direct the packet. The size of the IPv4 header ranges from 20 to 60 bytes. The first 20 bytes are a fixed set of information containing data such as the source and destination IP address, header length, and total length of the packet. The last set of bytes can range from 0 to 40 and consists of the options field.

The length of the data section of an IPv4 packet can vary greatly in size. However, the maximum possible size of an IPv4 packet is 65,535 bytes. It contains the message being transferred over the internet, like website information or email text. 

Diagram of an IPv4 packet header, 13 fields, and bit size
There are 13 fields within the header of an IPv4 packet:

Version (VER): This 4 bit component tells receiving devices what protocol the packet is using. The packet used in the illustration above is an IPv4 packet.

IP Header Length (HLEN or IHL): HLEN is the packet’s header length. This value indicates where the packet header ends and the data segment begins. 

Type of Service (ToS): Routers prioritize packets for delivery to maintain quality of service on the network. The ToS field provides the router with this information.

Total Length: This field communicates the total length of the entire IP packet, including the header and data. The maximum size of an IPv4 packet is 65,535 bytes.

Identification: IPv4 packets can be up to 65, 535 bytes, but most networks have a smaller limit. In these cases, the packets are divided, or fragmented, into smaller IP packets. The identification field provides a unique identifier for all the fragments of the original IP packet so that they can be reassembled once they reach their destination.

Flags: This field provides the routing device with more information about whether the original packet has been fragmented and if there are more fragments in transit.

Fragmentation Offset: The fragment offset field tells routing devices where in the original packet the fragment belongs.

Time to Live (TTL): TTL prevents data packets from being forwarded by routers indefinitely. It contains a counter that is set by the source. The counter is decremented by one as it passes through each router along its path. When the TTL counter reaches zero, the router currently holding the packet will discard the packet and return an ICMP Time Exceeded error message to the sender. 

Protocol: The protocol field tells the receiving device which protocol will be used for the data portion of the packet.

Header Checksum: The header checksum field contains a checksum that can be used to detect corruption of the IP header in transit. Corrupted packets are discarded.

Source IP Address: The source IP address is the IPv4 address of the sending device.

Destination IP Address: The destination IP address is the IPv4 address of the destination device.

Options: The options field allows for security options to be applied to the packet if the HLEN value is greater than five. The field communicates these options to the routing devices.

Difference between IPv4 and IPv6
In an earlier part of this course, you learned about the history of IP addressing. As the internet grew, it became clear that all of the IPv4 addresses would eventually be depleted; this is called IPv4 address exhaustion. At the time, no one had anticipated how many computing devices would need an IP address. IPv6 was developed to mitigate IPv4 address exhaustion and other related concerns. 

Some of the key differences between IPv4 and IPv6 include the length and the format of the addresses. IPv4 addresses are made up of four decimal numbers separated by periods, each number ranging from 0 to 255. Together the numbers span 4 bytes, and allow for up to 4.3 billion possible addresses. An example of an IPv4 address would be: 198.51.100.0. IPv6 addresses are made of eight hexadecimal numbers separated by colons, each number consisting of up to four hexadecimal digits. Together, all numbers span 16 bytes, and allow for up to 340 undecillion addresses (340 followed by 36 zeros). An example of an IPv6 address would be: 2002:0db8:0000:0000:0000:ff21:0023:1234.

Note: to represent one or more consecutive sets of all zeros, you can replace the zeros with a double colon "::", so the above IPv6 address would be "2002:0db8::ff21:0023:1234."

There are also some differences in the layout of an IPv6 packet header. The IPv6 header format is much simpler than IPv4. For example, the IPv4 Header includes the IHL, Identification, and Flags fields, whereas the IPv6 does not. The IPv6 header only introduces the Flow Label field, where the Flow Label identifies a packet as requiring special handling by other IPv6 routers. 

Side by side diagrams of an IPv4 packet header and a simplified IPv6 packet
There are some important security differences between IPv4 and IPv6. IPv6 offers more efficient routing and eliminates private address collisions that can occur on IPv4 when two devices on the same network are attempting to use the same address.

Overview of Network Protocols

A network protocol is a collection of rules and standards that devices use to communicate across a network. These protocols define how data should be structured, transmitted, received, and processed between connected systems.

When data travels across a network, the protocol information acts like a shared language between devices. It provides instructions that tell the receiving device how to interpret and handle the incoming data. Without protocols, devices from different manufacturers and systems would not be able to communicate effectively.

Network protocols are essential for internet communication, file transfers, email services, remote access, and many other network operations.

Although protocols are necessary for communication, they can also introduce security risks. Some protocols contain weaknesses that attackers may attempt to exploit. For example, cybercriminals may abuse the Domain Name System (DNS) protocol to redirect users from legitimate websites to harmful or fake websites that distribute malware or steal information.

Because of these risks, cybersecurity professionals must understand both the functionality and security implications of common network protocols.

Categories of Network Protocols

Network protocols are generally grouped into three major categories:

Communication Protocols
Management Protocols
Security Protocols

There are many networking protocols in use today, but understanding the most common ones is especially important for network administrators and entry-level security analysts.

Communication Protocols

Communication protocols control how information is exchanged between devices on a network. They define:

How data is transmitted
The timing of communication
Error handling methods
Recovery procedures for lost data

Below are some widely used communication protocols.

Transmission Control Protocol (TCP)

TCP is a transport-layer protocol that creates a reliable connection between two devices before data transmission begins.

TCP uses a process called the three-way handshake to establish a connection:

The client sends a SYN (synchronize) request
The server responds with a SYN/ACK acknowledgment
The client sends a final ACK response

After this process, the connection becomes active and data transfer can begin.

TCP focuses on reliability by:

Confirming packet delivery
Reordering packets correctly
Retransmitting missing data
Detecting transmission errors

In the TCP/IP model, TCP operates at the Transport Layer.

User Datagram Protocol (UDP)

UDP is also a transport-layer protocol, but unlike TCP, it does not establish a formal connection before sending data.

Because UDP skips connection setup and extensive error checking:

Data transmission is faster
Reliability is lower than TCP
Lost packets are usually not retransmitted

UDP is commonly used for applications that prioritize speed and real-time communication, such as:

Streaming services
Online gaming
DNS requests
Voice and video communication

UDP also operates at the Transport Layer of the TCP/IP model.

Hypertext Transfer Protocol (HTTP)

HTTP is an application-layer protocol used for communication between web browsers and website servers.

It allows clients to request web pages and receive website content from servers.

Key characteristics of HTTP include:

Uses port 80
Transfers website data without encryption
Considered insecure for sensitive communication

Because HTTP traffic is not encrypted, many websites now use HTTPS instead.

HTTP operates at the Application Layer.

Domain Name System (DNS)

DNS is responsible for translating domain names into IP addresses.

For example, when a user types a website name into a browser, a DNS server locates the corresponding IP address needed to access the website.

Important DNS details include:

Commonly uses UDP port 53
May switch to TCP when responses are large
Operates at the Application Layer

DNS is one of the most essential services on the internet because it allows users to access websites using readable names instead of numerical IP addresses.

Management Protocols

Management protocols are used to monitor, configure, and maintain network devices and performance.

These protocols help administrators manage network operations and troubleshoot problems efficiently.

Simple Network Management Protocol (SNMP)

SNMP is used to monitor and manage devices connected to a network.

With SNMP, administrators can:

Monitor bandwidth usage
Change device settings
Reset passwords
Modify network configurations
Collect device performance data

SNMP operates at the Application Layer of the TCP/IP model.

Internet Control Message Protocol (ICMP)

ICMP is used by networking devices to communicate error information and network status updates.

It helps identify problems related to:

Connectivity failures
Packet loss
Routing issues
Network latency

One common use of ICMP is the ping command, which tests whether a device is reachable across a network.

ICMP operates at the Internet Layer of the TCP/IP model.

Security Protocols

Security protocols are designed to protect data as it travels across networks. These protocols use encryption and secure communication methods to reduce the risk of unauthorized access or interception.

Hypertext Transfer Protocol Secure (HTTPS)

HTTPS is the secure version of HTTP.

It protects communication between web browsers and servers using SSL/TLS encryption.

Important HTTPS characteristics include:

Uses port 443
Encrypts transmitted data
Prevents attackers from easily reading network traffic
Improves website security and user privacy

HTTPS operates at the Application Layer.

Secure File Transfer Protocol (SFTP)

SFTP is a secure method for transferring files across networks.

It operates using Secure Shell (SSH) technology and commonly uses TCP port 22.

SFTP protects file transfers using strong encryption methods such as:

Advanced Encryption Standard (AES)
SSH encryption mechanisms

SFTP is widely used in:

Cloud storage systems
Secure remote file management
Enterprise file transfers

Whenever users upload or download files securely from cloud storage platforms, SFTP is often involved behind the scenes.

SFTP also operates at the Application Layer of the TCP/IP model.

Important Security Note

Although encryption protocols such as HTTPS and SFTP protect the contents of transmitted data, they do not completely hide all network information.

For example, attackers may still be able to view:

Source IP addresses
Destination IP addresses
Traffic patterns
Communication timing

This means that while encryption significantly improves confidentiality, some metadata about the communication can still be observed if network traffic is intercepted.







Network Address Translation (NAT)

Devices connected within a home or office network usually communicate using private IP addresses. These addresses are used only inside the local network and are not directly accessible from the public internet.

However, when devices on a Local Area Network (LAN) need internet access, they must communicate through a public IP address. Instead of assigning a separate public IP address to every device, the router uses a process called Network Address Translation (NAT).

NAT allows a router or firewall to:

Replace a device’s private IP address with the network’s public IP address for outgoing traffic
Convert the public address back to the correct private address when responses return

This enables multiple devices on the same local network to share a single public IP address while still communicating independently with the internet.

NAT typically operates across the:

Internet Layer
Transport Layer

of the TCP/IP model.

Private vs Public IP Addresses
Private IP Addresses

Private IP addresses are used inside local networks and are assigned by routers.

Characteristics of private IP addresses:

Used only within internal networks
Not directly reachable from the public internet
Free to use
Unique only inside the local network

Common private IPv4 address ranges include:

10.0.0.0−10.255.255.255

172.16.0.0−172.31.255.255

192.168.0.0−192.168.255.255

Public IP Addresses

Public IP addresses are globally unique addresses used for communication over the internet.

These addresses are:

Assigned through Internet Service Providers (ISPs)
Managed under internet authorities such as IANA
Reachable from anywhere on the internet
Usually leased or purchased by organizations and providers

Public IPv4 address ranges include nearly all address spaces outside the reserved private ranges.

Dynamic Host Configuration Protocol (DHCP)

Dynamic Host Configuration Protocol (DHCP) is a management protocol used to automatically configure devices on a network.

DHCP works with routers or DHCP servers to assign devices important network settings such as:

IP addresses
Default gateways
DNS server addresses
Subnet masks

Without DHCP, network administrators would need to manually configure each device individually.

DHCP uses:

UDP port 67 for DHCP servers
UDP port 68 for DHCP clients

DHCP operates at the Application Layer of the TCP/IP model.

Address Resolution Protocol (ARP)

ARP is used to translate IP addresses into MAC addresses within a local network.

Every network device has:

An IP address used for logical communication
A MAC address permanently assigned to its network interface card (NIC)

Although IP addresses may change, MAC addresses remain fixed for the hardware device.

When a device needs to communicate locally but only knows the target IP address, ARP helps identify the matching MAC address.

ARP functions include:

Mapping IP addresses to MAC addresses
Maintaining ARP cache tables
Supporting local network communication

ARP operates mainly at the Network Access Layer of the TCP/IP model.

Because ARP functions below the application layer, it does not use port numbers.

Telnet

Telnet is an application-layer protocol used for remote access and device management.

It allows users to connect to another system using command-line communication. However, Telnet sends all data in plain text, making it insecure for modern environments.

Characteristics of Telnet:

Uses TCP port 23
Does not encrypt traffic
Vulnerable to interception and credential theft

Because of its security weaknesses, Telnet has largely been replaced by SSH.

Secure Shell (SSH)

SSH is a secure protocol used for encrypted remote access to systems and devices.

SSH provides:

Secure authentication
Encrypted communication
Safer remote administration

Unlike Telnet, SSH protects transmitted data from interception.

Key SSH details:

Uses TCP port 22
Replaces insecure remote access protocols
Operates at the Application Layer

SSH is widely used by system administrators, cybersecurity teams, and cloud infrastructure managers.

Post Office Protocol (POP3)

POP3 is an email retrieval protocol used to download messages from a mail server to a local device.

When users refresh their email application, POP3 may retrieve newly received messages from the mail server.

Characteristics of POP3:

Downloads emails locally before reading
Messages may optionally be removed from the server after download
Limited synchronization across multiple devices

POP3 commonly uses:

TCP/UDP port 110 for unencrypted communication
TCP/UDP port 995 for encrypted communication using SSL/TLS

POP3 operates at the Application Layer.

Internet Message Access Protocol (IMAP)

IMAP is another protocol used for receiving email, but unlike POP3, it keeps messages stored on the mail server.

This allows users to access the same mailbox from multiple devices while maintaining synchronization.

Features of IMAP include:

Downloads email headers and content as needed
Supports multi-device synchronization
Allows partial reading before full download
Keeps mail stored remotely on the server

IMAP uses:

TCP port 143 for unencrypted communication
TCP port 993 for encrypted communication over TLS

IMAP also operates at the Application Layer.

Simple Mail Transfer Protocol (SMTP)

SMTP is used to send and route outgoing email messages between mail servers and recipients.

SMTP works with mail transfer systems to ensure messages reach the correct destination by resolving email server addresses through DNS.

SMTP functions include:

Sending outgoing mail
Routing email between servers
Supporting mail delivery infrastructure
Assisting spam control measures

SMTP commonly uses:

TCP/UDP port 25 for unencrypted email
TCP/UDP port 587 for encrypted communication using TLS

SMTP operates at the Application Layer.

Importance of Ports and Protocols

Port numbers help devices determine which service or application should process incoming data packets.

Firewalls and security systems often use port numbers to:

Allow trusted traffic
Block unwanted communication
Restrict access to sensitive services

For example, an organization may configure its firewall to only allow secure email traffic through encrypted POP3 port 995 from approved IP addresses.

Understanding protocols and port numbers is extremely important for cybersecurity professionals because they are frequently involved in:

Network analysis
Security monitoring
Incident investigations
Firewall configuration
Technical interviews

Introduction to Wireless Communication Protocols

Wireless internet access is commonly referred to as Wi-Fi. Wi-Fi is based on a group of communication standards used for wireless local area networks (WLANs). The term “Wi-Fi” was originally created by the Wireless Ethernet Compatibility Alliance (WECA), which later became known as the Wi-Fi Alliance.

The technical standards behind Wi-Fi are part of the IEEE 802.11 family of networking standards developed by the Institute of Electrical and Electronics Engineers (IEEE). Because of this, wireless networking technologies are often identified as IEEE 802.11 standards.

To protect wireless communication, different wireless security protocols have been developed over time. As older protocols became vulnerable to attacks, newer versions were introduced with stronger encryption and improved authentication methods.

This evolution moved from:

WEP
WPA
WPA2
WPA3

Each new version improved wireless security and addressed weaknesses found in earlier technologies.

Wired Equivalent Privacy (WEP)

Wired Equivalent Privacy (WEP) was the first widely used wireless security protocol and was introduced in 1999.

The goal of WEP was to provide wireless networks with a level of privacy similar to traditional wired networks.

Although WEP was important during the early growth of wireless networking, it is now considered outdated and insecure.

Key Characteristics of WEP
Designed for early wireless networks
Intended to secure Wi-Fi communication
Used encryption to protect wireless traffic
Became vulnerable due to weaknesses in the protocol itself

Today, WEP is rarely used because attackers can break its encryption relatively easily.

However, cybersecurity professionals may still encounter WEP in:

Older routers
Legacy devices
Outdated network environments

Because WEP contains major security flaws, it is considered a high-risk wireless protocol.

Wi-Fi Protected Access (WPA)

Wi-Fi Protected Access (WPA) was introduced in 2003 as an improvement over WEP.

WPA was designed to:

Fix WEP vulnerabilities
Improve wireless security
Maintain compatibility with older hardware devices

WPA introduced a stronger encryption system known as:

Temporal Key Integrity Protocol (TKIP)

TKIP improved security by:

Using larger encryption keys
Dynamically changing encryption information
Making brute-force attacks more difficult

WPA also introduced a message integrity check, which helps detect whether transmitted data has been altered during communication.

This feature helps prevent:

Packet tampering
Replay attacks
Unauthorized modification of transmissions
WPA Vulnerabilities

Although WPA improved wireless security significantly, it still contained weaknesses.

One major vulnerability is the:

KRACK Attack (Key Reinstallation Attack)

In a KRACK attack, attackers manipulate the WPA authentication handshake process and may force the reuse of encryption keys.

Under certain conditions, attackers can:

Intercept wireless traffic
Decrypt communications
Inject malicious data into transmissions

Because of these security concerns, WPA was eventually replaced by WPA2.

WPA2

WPA2 was released in 2004 and became the primary wireless security standard for Wi-Fi networks.

WPA2 improved wireless protection by replacing older encryption methods with:

Advanced Encryption Standard (AES)

AES provides much stronger encryption than earlier wireless security methods.

WPA2 also introduced:

Counter Mode Cipher Block Chaining Message Authentication Code Protocol (CCMP)

CCMP improves:

Data confidentiality
Message authentication
Data integrity

Due to these improvements, WPA2 became the standard security protocol for most wireless networks worldwide.

However, WPA2 is still vulnerable to KRACK attacks, which contributed to the development of WPA3.

WPA2 Modes

WPA2 includes two operating modes:

WPA2 Personal

WPA2 Personal is commonly used for home networks.

Characteristics include:

Easier setup process
Shared password across devices
Suitable for small environments

Because all devices use the same wireless passphrase, managing large numbers of users becomes difficult in enterprise environments.

WPA2 Enterprise

WPA2 Enterprise is designed for businesses and organizations.

It provides:

Centralized authentication
Individual user access management
Better administrative control
Stronger enterprise security

In enterprise mode:

Users do not directly access encryption keys
Administrators can easily add or remove network access
Authentication is managed through centralized systems

Although setup is more complex than WPA2 Personal, it provides much stronger organizational security.

WPA3

WPA3 was introduced in 2018 as the newest generation of Wi-Fi security protocols.

It was specifically developed to address vulnerabilities found in WPA2 and improve wireless protection overall.

Major Improvements in WPA3
Protection Against KRACK Attacks

WPA3 improves the authentication process and reduces the risk of key reinstallation attacks that affected WPA2.

Simultaneous Authentication of Equals (SAE)

WPA3 replaces older authentication methods with:

Simultaneous Authentication of Equals (SAE)

SAE improves password security by:

Protecting authentication exchanges
Preventing offline password-guessing attacks
Making it more difficult for attackers to capture and crack wireless credentials

This mechanism prevents attackers from downloading authentication data and attempting to decode passwords later.

Stronger Encryption

WPA3 increases encryption strength for wireless communication.

Key encryption improvements include:

Standard 128-bit encryption
Optional 192-bit encryption in WPA3-Enterprise mode

These enhancements improve protection for sensitive business and enterprise environments.

Evolution of Wireless Security Protocols

The development of wireless security protocols reflects the ongoing effort to improve Wi-Fi protection against evolving cyber threats.

Protocol	Release Year	Main Features	Security Status
WEP	1999	Basic wireless encryption	Insecure and outdated
WPA	2003	TKIP encryption, integrity checks	Vulnerable to KRACK
WPA2	2004	AES encryption, CCMP	Strong but still vulnerable to KRACK
WPA3	2018	SAE authentication, stronger encryption	Most secure modern standard

Overview of Subnetting

Subnetting is the practice of dividing a large network into multiple smaller sections called subnets. Each subnet functions as its own smaller network within the larger overall network.

A simple way to understand subnetting is to imagine a large city divided into separate neighborhoods. The entire city represents the main network, while each neighborhood represents an individual subnet.

Subnetting helps organize network traffic more efficiently by grouping devices into smaller sections. Every subnet is identified through a combination of:

An IP address
A subnet mask (or network prefix)

This combination defines which devices belong to a particular subnet.

Purpose of Subnetting

Subnetting is commonly used to improve:

Network organization
Communication efficiency
Bandwidth management
Security segmentation

When devices communicate within the same subnet, the traffic usually remains inside that subnet instead of traveling across the entire network. This reduces unnecessary traffic and improves overall network performance.

For example:

Devices in one department may communicate within one subnet
Another department may use a separate subnet
Routers manage communication between these subnetworks

This structure allows networks to operate more efficiently and securely.

Classless Inter-Domain Routing (CIDR)

Modern subnetting commonly uses Classless Inter-Domain Routing (CIDR).

CIDR is a method of assigning subnet masks to IP addresses in a more flexible and efficient way than older addressing systems.

CIDR replaced the older classful addressing model that grouped IP addresses into fixed classes:

Class A
Class B
Class C
Class D
Class E

The older class-based system became inefficient as the number of internet-connected devices rapidly increased during the 1990s. CIDR was introduced to make better use of available IPv4 addresses.

CIDR Notation

CIDR notation combines an IP address with a slash (/) followed by a number known as the network prefix.

Example:

198.51.100.0/24

In this example:

198.51.100.0 represents the network address
/24 represents the network prefix

The /24 indicates that the first 24 bits identify the network portion of the address, while the remaining bits identify individual devices within that subnet.

This subnet typically includes addresses ranging from:

198.51.100.0−198.51.100.255

CIDR allows network administrators to divide networks into smaller, manageable subnetworks without being limited by rigid address classes.

Advantages of CIDR

CIDR offers several important benefits:

More efficient use of IPv4 addresses
Reduced size of routing tables
Improved flexibility in network design
Better scalability for growing networks

By allowing variable subnet sizes, CIDR helps organizations allocate only the number of IP addresses they actually need.

Security Benefits of Subnetting

Subnetting is also valuable for improving network security.

By separating devices into different subnetworks, organizations can:

Create isolated security zones
Reduce unnecessary network exposure
Limit the spread of attacks
Control communication between departments or systems

For example:

Employee devices may exist on one subnet
Servers may exist on another subnet
Guest Wi-Fi users may use a completely separate subnet

Firewalls and routing rules can then regulate communication between these subnetworks.

This layered approach improves both security and traffic management.

Network Efficiency and Performance

Subnetting improves network performance by reducing broadcast traffic and limiting unnecessary communication across the entire network.

Benefits include:

Faster local communication
Better bandwidth usage
Reduced congestion
Improved troubleshooting and management

When devices communicate within the same subnet, switches can keep traffic localized rather than forwarding it across multiple network segments.

Common Network Protocols

Network protocols are standardized rules that allow devices on a network to communicate with each other correctly. These protocols determine how information is transmitted, received, interpreted, and routed between devices and services across a network.

Protocols ensure that different devices and systems can exchange data using a shared communication structure, regardless of hardware or software differences.

Network protocols are generally grouped into three major categories:

Communication Protocols
Management Protocols
Security Protocols
Communication Protocols

Communication protocols are responsible for establishing and maintaining data exchange between systems and applications.

Some common communication protocols include:

Transmission Control Protocol (TCP)

TCP is a transport-layer protocol that creates a reliable connection between two devices before transmitting data.

TCP ensures:

Reliable delivery
Error checking
Packet reordering
Retransmission of lost data

It is commonly used for services that require accurate and complete communication.

User Datagram Protocol (UDP)

UDP is a connectionless transport protocol that sends data without establishing a dedicated connection first.

UDP prioritizes speed and efficiency over reliability, making it useful for:

Streaming services
Voice and video communication
Real-time applications
Simple Mail Transfer Protocol (SMTP)

SMTP is a communication protocol used to send and route email messages across networks.

SMTP provides the framework for transferring outgoing email between mail servers and recipient systems.

Management Protocols

Management protocols help monitor, troubleshoot, and manage network operations.

Internet Control Message Protocol (ICMP)

ICMP is commonly used for reporting network errors and testing connectivity between devices.

Functions of ICMP include:

Diagnosing network issues
Reporting unreachable systems
Measuring latency
Supporting tools such as ping

ICMP plays an important role in network troubleshooting and performance monitoring.

Security Protocols

Security protocols are designed to protect data as it moves across networks by using encryption and authentication methods.

Examples include:

Internet Protocol Security (IPSec)

IPSec secures network communication by encrypting IP traffic and authenticating transmitted data.

It is widely used in VPN technologies and secure communications between networks.

SSL/TLS

Secure Sockets Layer (SSL) and Transport Layer Security (TLS) provide encrypted communication between systems, especially for websites and online services.

TLS is the modern replacement for SSL and is commonly used in HTTPS connections.

Other Commonly Used Protocols
HyperText Transfer Protocol (HTTP)

HTTP is an application-layer protocol that allows communication between web browsers and web servers.

It enables users to:

Access websites
Request web pages
Receive online content

HTTP itself does not encrypt transmitted data.

Domain Name System (DNS)

DNS is an application-layer protocol responsible for translating domain names into IP addresses.

For example, DNS converts a website name into the numerical address needed to locate the server hosting the site.

Without DNS, users would need to remember numerical IP addresses instead of readable website names.

Address Resolution Protocol (ARP)

ARP is used to map IP addresses to physical MAC addresses on a local network.

When devices communicate within the same LAN, ARP helps identify the correct hardware destination associated with an IP address.

Over time, wireless security standards have evolved to improve encryption and reduce vulnerabilities.

These protocols use:

Advanced Encryption Standard (AES)

Personal Mode
Best suited for home networks
Uses shared passwords
Enterprise Mode
Designed for businesses and organizations

Network Security Tools and Practices

Organizations use several technologies to secure networks and protect data traffic.

Firewalls

Firewalls are security devices or software systems that monitor and filter network traffic entering or leaving a network.

They inspect data packets and determine whether traffic should be allowed or blocked based on predefined security rules.

Traditional firewall rules are commonly based on:

IP addresses
Port numbers
Protocol types
Types of Firewalls
Stateless Firewalls

Stateless firewalls filter traffic using fixed rules without remembering previous network activity.

Characteristics include:

Evaluates packets individually
Does not track active sessions
Requires separate rules for incoming and outgoing traffic
Stateful Firewalls

Stateful firewalls track active network connections using a state table.

Advantages include:

Monitors ongoing sessions
Automatically recognizes return traffic
Requires fewer rule configurations
Provides stronger traffic awareness

Because they understand connection states, stateful firewalls offer improved security compared to stateless firewalls.

Next-Generation Firewalls (NGFWs)

Next-generation firewalls provide advanced security features beyond traditional firewall protection.

NGFW capabilities may include:

Deep packet inspection
Intrusion prevention systems (IPS)
Application awareness
Malware sandboxing
Network antivirus
URL filtering
DNS filtering

Unlike traditional firewalls that mainly filter traffic by ports and IP addresses, NGFWs can identify and control traffic based on specific applications.

NGFWs also inspect traffic at the Application Layer of the TCP/IP model.

Proxy Servers

A proxy server acts as an intermediary between internal users and external networks.

Proxy servers often use:

Network Address Translation (NAT)

to help conceal internal network structures and provide an additional security layer.

Forward Proxy

A forward proxy handles requests from internal users attempting to access external resources on the internet.

Reverse Proxy

A reverse proxy handles incoming requests from external systems attempting to access internal servers or services.

Proxy servers can also:

Filter malicious websites
Block unwanted content
Enforce access policies
Improve privacy and monitoring
Virtual Private Networks (VPNs)

A Virtual Private Network (VPN) secures internet communication by encrypting data traffic and hiding a user’s IP address.

VPNs use a method called:

Encapsulation

Encapsulation wraps unencrypted data inside encrypted packets before transmission over public networks.

This helps protect data confidentiality and user privacy.

Benefits of VPNs

VPNs help organizations and individuals:

Secure sensitive communications
Protect remote access connections
Hide IP addresses
Reduce tracking and surveillance risks
Safeguard data on public networks

Businesses often use VPNs to connect employees securely to internal systems, applications, and corporate servers.

Software-Defined Wide Area Network (SD-WAN)

Many organizations now combine VPN technology with:

Software-Defined Wide Area Networks (SD-WAN)

SD-WAN is a virtual networking technology that securely connects users, applications, and branch locations across wide geographic regions.

Benefits of SD-WAN include:

Improved network flexibility
Centralized management

SD-WAN has become increasingly important in modern enterprise networking environments.










Virtual Private Networks (VPNs)

A Virtual Private Network (VPN) is a security technology that helps protect internet activity by hiding a user’s public IP address and encrypting data transmitted over the internet.

VPNs create a secure connection between a device and a network, especially when users are connected through public or untrusted networks such as public Wi-Fi or the internet.

A VPN works by routing internet traffic through a secure server that acts as an intermediary between the user’s device and the internet. This process creates a protected virtual tunnel that:

Conceals the user’s real IP address
Encrypts data while it is in transit
Helps maintain privacy and confidentiality online

The primary goal of a VPN is to establish a secure communication channel between systems and networks.

VPNs are especially important because they allow trusted and encrypted communication even across networks that are not inherently secure.

VPN Protocols

VPNs rely on specialized communication standards known as VPN protocols.

A VPN protocol is a set of rules that determines:

How secure tunnels are created
How data is encrypted
How information travels between connected devices

In networking, any connected device is called an endpoint. Examples of endpoints include:

Computers
Smartphones
Tablets
Servers

Different VPN providers support different VPN protocols depending on performance, compatibility, and security requirements.

Types of VPNs

VPNs are commonly divided into two major categories:

Remote Access VPNs
Site-to-Site VPNs
Remote Access VPNs

Remote access VPNs are typically used by individual users who need secure access to a private network from a personal device.

These VPNs create an encrypted connection between:

A user’s device
A remote VPN server

The connection is established over the internet and protects the user’s transmitted data from interception.

Remote access VPNs are commonly used for:

Working remotely
Secure browsing
Protecting public Wi-Fi sessions
Accessing organizational resources securely
Site-to-Site VPNs

Site-to-site VPNs are commonly used by organizations to securely connect multiple office networks across different locations.

These VPNs create encrypted tunnels between entire networks rather than individual devices.

For example:

A company headquarters may securely connect with branch offices
Multiple geographic locations can communicate over secure connections

Site-to-site VPNs are widely used in enterprise environments where organizations need secure communication across large distances.

One commonly used protocol in site-to-site VPNs is:

Internet Protocol Security (IPSec)

IPSec helps encrypt and authenticate traffic moving between connected networks.

Challenges of Site-to-Site VPNs

Although site-to-site VPNs provide strong security, they can also be:

Network Interception Attacks

Network interception attacks occur when attackers capture or interfere with data as it travels across a network. Their objective is often to steal sensitive information, monitor communications, or manipulate transmitted data.

Cybercriminals may use specialized hardware or software tools to examine network traffic, a technique commonly called packet sniffing. Through this method, attackers can observe information being exchanged between devices, even when they are not authorized to access it. In some cases, attackers go beyond monitoring and actively modify the intercepted data before it reaches its destination.

These attacks can seriously disrupt network operations. Attackers may inject malicious code, alter messages, or interrupt communication between systems. For example, during an online banking transaction, a malicious actor could intercept the transfer request and replace the recipient’s bank account details with their own account information.

Additional forms of interception-based attacks include on-path attacks and replay attacks, both of which exploit weaknesses in communication processes to compromise data security.

Backdoor Attacks

Backdoor attacks are another major threat that security professionals must understand. Consider a secure office building that uses cameras, access cards, and biometric scanners to control entry and exit. If someone discovers an unmonitored entrance and uses it to bypass security checks, they gain unauthorized access without detection.

In cybersecurity, a backdoor functions in a similar way. A backdoor is a hidden method of bypassing standard authentication or access controls within a system, application, or network. Sometimes developers or administrators intentionally create backdoors to simplify troubleshooting or maintenance tasks. However, these hidden access points can become dangerous if discovered or exploited by attackers.

Hackers may also install their own backdoors after compromising a system. This allows them to maintain long-term access even after the initial vulnerability is fixed. Once inside the network, attackers can carry out additional malicious activities such as installing malware, stealing confidential information, launching denial of service (DoS) attacks, or weakening existing security settings.

A denial of service attack is designed to overwhelm a server or network with excessive traffic, preventing legitimate users from accessing services or resources.

Potential Effects on Organizations

Cyberattacks can create severe consequences for organizations in several areas, including finances, reputation, and public safety.

Financial Consequences

Attacks such as DoS incidents can force systems offline, interrupting business operations and preventing organizations from generating revenue. For large companies, even short periods of downtime can result in significant financial losses. Organizations may also face high recovery expenses related to rebuilding infrastructure, restoring systems, or paying ransomware demands.

If attackers gain access to customer or client information, companies may additionally face lawsuits, regulatory penalties, and compensation claims, all of which can further increase financial damage.

Reputational Damage

A cyberattack can weaken public trust in an organization. When news of a security breach becomes public, customers may question whether the company can adequately protect sensitive information. As confidence declines, customers may decide to move to competing businesses that appear more secure and reliable.

Risks to Public Safety

Cyberattacks targeting government agencies or critical infrastructure can create risks beyond financial loss. If attackers compromise systems connected to power grids, water treatment facilities, transportation systems, or military communications, the effects could directly impact public safety and national security.

Because of these risks, governments and defense organizations worldwide continue investing heavily in cybersecurity measures to defend against increasingly advanced cyber threats.

Network Protocol Analyzers

A network protocol analyzer, also known as a packet analyzer or packet sniffer, is a tool used to capture, inspect, and analyze data moving across a network. These tools are widely used by cybersecurity professionals and network administrators to monitor traffic, troubleshoot problems, and investigate suspicious activity.

There are many network protocol analyzers available today. Some commonly used tools include:

SolarWinds NetFlow Traffic Analyzer
ManageEngine OpManager
Azure Network Watcher
Wireshark
tcpdump

This discussion mainly focuses on tcpdump, although many of the same concepts apply to other protocol analyzers used in cybersecurity. These tools help analysts investigate network intrusions, detect unusual traffic patterns, and identify attacks such as denial of service (DoS) incidents.

tcpdump

tcpdump is a lightweight, command-line-based network protocol analyzer commonly used on Unix and Linux systems. It relies on the open-source libpcap library to capture packets traveling through a network.

Because tcpdump operates entirely in the terminal, users interact with it through text commands rather than a graphical interface. It is preinstalled on many Linux distributions and can also run on other Unix-based operating systems, including macOS.

The tool provides a concise summary of captured network packets, presenting important traffic details in a readable format. Information displayed by tcpdump often includes:

Source IP address
Destination IP address
Port numbers used in communication
Protocol information
Packet transmission details

This makes tcpdump useful for quickly examining live network activity directly from the command line.

Understanding tcpdump Output

After a capture command is executed, tcpdump displays packet information directly in the terminal and can optionally save the output into a log file for later analysis.

A packet capture generated by tcpdump typically contains several important fields:

Timestamp

Each packet entry begins with a timestamp showing when the packet was captured. The format usually includes hours, minutes, seconds, and fractions of a second.

Source IP Address

This identifies the original sender of the packet.

Source Port

The source port indicates the port number from which the packet originated.

Destination IP Address

This field identifies the intended recipient of the packet.

Destination Port

The destination port specifies the port number receiving the packet.

By default, tcpdump may also translate IP addresses into hostnames and replace certain port numbers with the names of commonly associated services.

Common Applications of Network Protocol Analyzers

Tools like tcpdump are frequently used to observe network communications and evaluate network performance. They also help organizations maintain visibility into network activity and security conditions.

Some common uses include:

Creating baselines for normal network traffic behavior
Monitoring network utilization and performance metrics
Detecting suspicious or malicious communications
Identifying potential security incidents
Configuring custom alerts for network or security issues
Discovering unauthorized messaging traffic or rogue wireless access points

These capabilities make protocol analyzers valuable tools for cybersecurity investigations and network troubleshooting.

Misuse by Attackers

Although network analyzers are important defensive tools, attackers can also misuse them for malicious purposes. Cybercriminals may capture packets containing sensitive information such as usernames, passwords, or confidential communications.

For this reason, cybersecurity analysts must understand both the legitimate uses and potential risks associated with packet analysis tools. Knowledge of how these analyzers work helps security teams better detect threats and protect network communications from unauthorized monitoring.







DDoS Attack on a Major DNS Service Provider

DNS (Domain Name System) servers play a key role in how the internet works. They convert human-readable website addresses (like a URL) into numerical IP addresses that computers use to locate web servers. In simple terms, when a user enters a website in a browser, the DNS system ensures the request is routed to the correct destination server.

In this case study, a large DNS service provider that supported multiple organizations became the target of a Distributed Denial of Service (DDoS) attack on October 21, 2016.

Background of the Incident

Before the attack, a group of university students had developed a botnet intended for attacking gaming servers and online services. A botnet is a network of compromised devices infected with malware and controlled remotely by a single operator (often called a bot-herder). Each infected device (or “bot”) can be instructed to send traffic or requests to a chosen target.

The creators of this botnet publicly shared its code online. This made it accessible to a large number of users, including malicious actors who later repurposed it for larger-scale attacks. As a result, the botnet was eventually used against the DNS service provider.

How the Attack Happened

On the morning of the incident, at around 7:00 a.m., the attackers launched a massive flood of DNS requests toward the provider’s infrastructure. The botnet generated tens of millions of requests in a very short period of time.

Because DNS services are critical for routing internet traffic, this sudden overload caused the system to become unresponsive. As a result, websites relying on the affected DNS provider could no longer be reached, even though their actual servers were still online.

The impact was widespread, causing disruptions across multiple regions, including parts of North America and Europe, as users were unable to resolve domain names into IP addresses.

Impact and Recovery

The attack led to a significant outage lasting around two hours before services were restored. During this period, many online platforms experienced downtime because users could not access their websites through normal domain names.

Although additional waves of botnet traffic were launched afterward, the DNS provider had already strengthened its defenses and was able to reduce the impact of further attacks.

Understanding Packet Sniffing

Packet sniffing is the process of capturing and analyzing data packets as they move across a network. In a normal network environment, packets are delivered only to the intended destination device. Each device uses a Network Interface Card (NIC), which is the hardware responsible for connecting the system to the network and handling incoming traffic. The NIC checks whether the packet contains the device’s MAC address before accepting and processing it according to the required protocol.

A NIC can also operate in promiscuous mode, where it accepts all network traffic instead of only packets addressed to its device. Attackers may use tools such as Wireshark to capture network traffic in this mode. The collected information can later be used to steal sensitive data or imitate trusted users on the network through spoofing techniques.

Understanding IP Spoofing

After monitoring network traffic, attackers can impersonate legitimate systems by falsifying IP or MAC address information. This technique is known as IP spoofing. By pretending to be an authorized device, attackers may bypass trust relationships within a network.

One way organizations defend against spoofing is by configuring firewalls to block suspicious or unauthorized packets. Monitoring unusual traffic patterns is also important for detecting these attacks early.

On-Path Attacks

An on-path attack occurs when a malicious actor secretly places themselves between two trusted devices or systems that are communicating with each other. The attacker can intercept sensitive information such as login credentials, session data, or confidential communications. This type of attack is also commonly called a “man-in-the-middle” attack.

Attackers may also target DNS requests. By intercepting a DNS lookup, they can send a fake response that redirects users to a harmful or fraudulent website instead of the legitimate destination.

Encrypting data while it is traveling across the network, such as by using TLS, is one of the most effective protections against on-path attacks because it prevents attackers from easily reading or modifying intercepted traffic.

Smurf Attacks

A smurf attack is a form of network flooding attack that combines IP spoofing with denial-of-service techniques. In this attack, the attacker spoofs the victim’s IP address and sends large numbers of ICMP requests to a network broadcast address. The broadcast system then forwards the requests to many devices on the network at once.

As all systems reply simultaneously, the victim becomes overwhelmed with traffic, which can slow down or completely disable network services. Because ICMP is commonly used for diagnostics and troubleshooting, excessive ICMP responses can heavily impact servers and network performance.

Modern firewalls, especially Next Generation Firewalls (NGFWs), help defend against smurf attacks by identifying abnormal traffic patterns and blocking unusually large broadcasts before they disrupt operations.

Denial of Service (DoS) Attacks

A Denial of Service (DoS) attack aims to make a system or service unavailable to legitimate users. Attackers overwhelm the target with excessive traffic or requests until the server can no longer respond properly.

Unlike spoofing attacks where fake addresses are used to impersonate users, DoS attacks may involve traffic that appears legitimate. However, attackers often continue sending massive numbers of malicious packets until the target system crashes or becomes unresponsive.

Organizations reduce the risk of DoS attacks by applying layered security measures such as firewalls, traffic filtering, rate limiting, and encryption technologies.

Defense-in-Depth Strategy

No single security control can stop every attack. A stronger security posture is achieved by combining multiple layers of defense. Firewalls, encryption, traffic monitoring, intrusion detection systems, and secure network configurations all work together to reduce the chances of successful attacks and improve overall network resilience.
