### Understanding Variables in Python

A **variable** is a named location in memory used to store information. It acts like a container that holds a value, which can be accessed and modified throughout a program. Variables can store different kinds of data, such as numbers, text, or logical values.

A useful way to think about a variable is as a labeled box. The label (variable name) stays the same, but the item inside the box (value) can be replaced whenever needed.

In cybersecurity and programming tasks, variables may be used to store information such as user IDs, login attempt counts, IP addresses, or access control lists.

---

## Creating and Updating Variables

### Assigning a Value

To create a variable, place the variable name on the left side of the equals sign (`=`) and the value on the right side.

```python
username = "nzhao"
```

Here, the variable `username` stores the text value `"nzhao"`.

### Reassigning a Value

The value stored in a variable can be changed by assigning a new value to the same variable name.

```python
username = "zhao2"
```

Although the stored value changes, the variable name `username` remains unchanged.

**Note:** Text values must be enclosed in quotation marks because they are treated as strings. Python automatically determines the data type of a variable based on the assigned value.

---

## Assigning One Variable to Another

A variable can also receive the value stored in another variable.

```python
username = "nzhao"
old_username = username
```

After execution, both `username` and `old_username` contain the value `"nzhao"`.

---

## Example: Tracking Username Changes

The following example saves the original username before updating it:

```python
username = "nzhao"
old_username = username

username = "zhao2"

print("Previous username:", old_username)
print("Current username:", username)
```

**Output:**

```python
Previous username: nzhao
Current username: zhao2
```

This approach is useful when you need to preserve previous information before making updates.

---

# Conditional Statements in Python

## What Are Conditional Statements?

A **conditional statement** allows a program to make decisions based on whether a specified condition is true or false. The condition is evaluated, and the program executes different actions depending on the result.

* If the condition evaluates to **True**, the associated code runs.
* If the condition evaluates to **False**, the associated code is skipped or an alternative action is performed.

Conditions are commonly created by comparing values using comparison operators.

---

## Comparison Operators

| Operator | Meaning                  |
| -------- | ------------------------ |
| `>`      | Greater than             |
| `<`      | Less than                |
| `>=`     | Greater than or equal to |
| `<=`     | Less than or equal to    |
| `==`     | Equal to                 |
| `!=`     | Not equal to             |

These operators can be used with numbers and, in many cases, strings.

### Example

```python
status == 200
```

This expression checks whether `status` has a value of `200`.

---

# The `if` Statement

The `if` statement is the foundation of decision-making in Python. It evaluates a condition and executes code only when the condition is true.

### Syntax

```python
if status == 200:
    print("OK")
```

In this example:

* `if status == 200:` is the header.
* `print("OK")` is the body.

If `status` equals `200`, the message `"OK"` is displayed.

---

## Components of an `if` Statement

### 1. Header

The header contains:

* The keyword `if`
* A condition
* A colon (`:`) at the end

```python
if status == 200:
```

Parentheses around the condition are optional:

```python
if (status == 200):
```

Both versions work the same way.

### 2. Body

The body contains the instructions that run when the condition is true.

```python
if status == 200:
    print("OK")
```

Python uses **indentation** to identify code that belongs to the body.

---

# The `else` Statement

The `else` block provides an alternative action when the `if` condition evaluates to false.

### Example

```python
if status == 200:
    print("OK")
else:
    print("Check other status")
```

### How It Works

* If `status` is `200`, Python prints `"OK"`.
* Otherwise, it prints `"Check other status"`.

---

# The `elif` Statement

When multiple conditions need to be checked, use `elif` (short for "else if").

### Example

```python
if status == 200:
    print("OK")

elif status == 400:
    print("Bad Request")

elif status == 500:
    print("Internal Server Error")
```

### Execution Flow

Python checks conditions from top to bottom:

1. Is `status` equal to `200`?
2. If not, is it `400`?
3. If not, is it `500`?

As soon as a condition evaluates to **True**, Python executes that block and skips the remaining `elif` statements.

---

## Combining `elif` with `else`

An `else` block can be added to handle all remaining cases.

```python
if status == 200:
    print("OK")

elif status == 400:
    print("Bad Request")

elif status == 500:
    print("Internal Server Error")

else:
    print("Check other status")
```

This ensures that a message is displayed regardless of the status code.

---

# Logical Operators

Logical operators allow multiple conditions to be combined into a single expression.

---

## `and` Operator

The `and` operator requires **all conditions** to be true.

### Example

```python
if status >= 200 and status <= 226:
    print("Successful response")
```

### Explanation

The message is printed only when:

* `status` is greater than or equal to `200`
* **AND**
* `status` is less than or equal to `226`

Both conditions must be true.

---

## `or` Operator

The `or` operator requires **at least one condition** to be true.

### Example

```python
if status == 100 or status == 102:
    print("Informational response")
```

### Explanation

The message appears when:

* `status` is `100`
* **OR**
* `status` is `102`

Only one condition needs to be satisfied.

---

## `not` Operator

The `not` operator reverses a condition's result.

* True becomes False
* False becomes True

### Example

```python
if not(status >= 200 and status <= 226):
    print("Check status")
```

### Explanation

Python first evaluates:

```python
status >= 200 and status <= 226
```

Then `not` reverses the result.

The message is printed when the status code falls **outside** the range `200–226`.

---

## Why Parentheses Matter with `not`

Parentheses ensure Python evaluates the entire expression before applying `not`.

```python
if not(status >= 200 and status <= 226):
```

Without parentheses, Python may interpret the condition differently, leading to unexpected results.

---

# Key Points to Remember

* Conditional statements help programs make decisions.
* `if` executes code when a condition is true.
* `else` provides an alternative when all previous conditions are false.
* `elif` allows multiple conditions to be checked sequentially.
* Comparison operators are used to create conditions.
* `and` requires all conditions to be true.
* `or` requires at least one condition to be true.
* `not` reverses the outcome of a condition.
* Colons (`:`) and proper indentation are essential for correct Python syntax.
* Once an `elif` condition evaluates to true, the remaining `elif` conditions are skipped.

# Iterative Statements and Loops in Python

## Introduction to Loops

An **iterative statement**, commonly called a **loop**, allows a program to execute the same block of code repeatedly. Depending on the condition or sequence being processed, a loop may run multiple times or not run at all.

Python provides two primary loop types:

* **for loops** – used when iterating through a sequence.
* **while loops** – used when repetition depends on a condition.

Additionally, Python offers the **break** and **continue** statements to control loop behavior.

---

# For Loops

A **for loop** is used when you want to process each item in a sequence such as a list, tuple, string, or range of numbers.

### Basic Example

```python
for user in ["elarson", "bmoreno", "tshah", "sgilmore"]:
    print(user)
```

### How It Works

The loop consists of two parts:

#### Loop Header

```python
for user in ["elarson", "bmoreno", "tshah", "sgilmore"]:
```

* `for` begins the loop.
* `user` is the loop variable.
* `in` indicates the sequence being processed.
* The header ends with a colon (`:`).

#### Loop Body

```python
print(user)
```

The indented code executes once for each item in the sequence.

---

# Iterating Through Lists

Lists are commonly used with for loops.

### Example

```python
computer_assets = ["laptop1", "desktop20", "smartphone03"]

for asset in computer_assets:
    print(asset)
```

### Output

```text
laptop1
desktop20
smartphone03
```

During each iteration, `asset` stores one item from the list.

---

# Iterating Through Strings

A string can also be treated as a sequence.

### Example

```python
word = "security"

for character in word:
    print(character)
```

### Output

```text
s
e
c
u
r
i
t
y
```

Each loop iteration processes one character.

---

# Using `range()`

The `range()` function generates a sequence of numbers that can be used in a for loop.

### Syntax

```python
range(start, stop, step)
```

Parameters:

* **start** – first number included
* **stop** – ending value (not included)
* **step** – amount to increase each iteration

### Example

```python
for i in range(0, 5, 1):
    print(i)
```

### Output

```text
0
1
2
3
4
```

Notice that `5` is excluded.

---

## Simplified Range

* Start = 0
* Step = 1

they can be omitted.

```python

    print(i)
```

Produces the same result:

```text

1
2
3
4
```

# While Loops

A **while loop** continues running as long as its condition remains true.

### Example

```python
i = 1

while i < 5:
    print(i)
    i = i + 1
```

### Output

```text
1
2
3
4
```

### How It Works

1. `i` starts at 1.
2. Python checks if `i < 5`.
3. If true, the loop executes.
4. `i` increases by 1.
5. The process repeats until the condition becomes false.

---

# Using Integers in While Conditions

A common use case is tracking counts.

### Example: Login Attempts

```text
Login attempts: 3
```

The loop stops when `login_attempts` reaches 5.

While loops can also depend on Boolean values.

```python
    if count == 4:
```


The loop ends once `login_status` becomes `False`.

---

# Controlling Loops

Python provides two special statements for managing loop execution:

* `break`
* `continue`

---

# The `break` Statement

The `break` statement immediately terminates a loop.

### Example

```python
computer_assets = ["laptop1", "desktop20", "smartphone03"]

for asset in computer_assets:
    if asset == "desktop20":
        break

    print(asset)
```

### Output

```text
laptop1
```

### Explanation

* First iteration: prints `"laptop1"`.
* Second iteration: encounters `"desktop20"`.
* `break` stops the loop immediately.
* Remaining items are never processed.

---

# The `continue` Statement

The `continue` statement skips the current iteration and moves to the next one.

### Example

```python
computer_assets = ["laptop1", "desktop20", "smartphone03"]

for asset in computer_assets:
    if asset == "desktop20":
        continue

    print(asset)
```

### Output

```text
laptop1
smartphone03
```

### Explanation

* `"desktop20"` is skipped.
* The loop continues processing the remaining items.

---

# `break` vs `continue`

| Statement  | Effect                                            |
| ---------- | ------------------------------------------------- |
| `break`    | Stops the loop completely                         |
| `continue` | Skips the current iteration and continues looping |

### Example

```python
for number in range(5):

    if number == 2:
        break

    print(number)
```

Output:

```text
0
1
```

---

```python
for number in range(5):

    if number == 2:
        continue

    print(number)
```

Output:

```text
0
1
3
4
```

---

# Infinite Loops

An **infinite loop** occurs when a loop never reaches a stopping condition.

### Example

```python
while True:
    print("Running...")
```

Since the condition is always true, the loop continues indefinitely.

Infinite loops may be intentional for long-running services, but they are often caused by programming mistakes, such as forgetting to update a loop variable.

To stop an infinite loop manually:

* **Windows:** `Ctrl + C`
* **Linux/macOS:** `Ctrl + C`
* Some environments may also support `Ctrl + Z`

---

# Key Takeaways

* Loops execute code repeatedly.
* Use **for loops** when working through sequences.
* Use **while loops** when repetition depends on a condition.
* The `range()` function generates numerical sequences.
* Lists and strings can both be iterated through using for loops.
* `break` exits a loop immediately.
* `continue` skips the current iteration and proceeds to the next.
* Loop bodies must be properly indented.
* Infinite loops occur when the termination condition is never reached.

# Functions in Cybersecurity (Reframed Notes)

## What is a Function?

A **function** is a reusable block of code that performs a specific task. Functions help automate repetitive activities and make programs easier to read, maintain, and update.

In cybersecurity, analysts often repeat the same tasks, such as:

* Analyzing security logs
* Detecting failed login attempts
* Searching for suspicious activity
* Generating security alerts

Instead of writing the same code multiple times, a function can be created once and reused whenever needed.

---

## Types of Functions in Python

### 1. Built-in Functions

Functions that already exist in Python and can be used directly.

**Example:**

```python
print("Hello")
```

Here, `print()` is a built-in function.

### 2. User-Defined Functions

Functions created by programmers to perform specific tasks.

**Example:**

```python
def display_investigation_message():
    print("investigate activity")
```

---

## Defining a Function

A function definition consists of:

### Function Header

The header tells Python that a function is being created.

**Syntax:**

```python
def function_name():
```

**Example:**

```python
def display_investigation_message():
```

Components:

* `def` → Keyword used to define a function
* `display_investigation_message` → Function name
* `()` → Parentheses required by syntax
* `:` → Marks the start of the function body

---

## Function Body

The function body contains the instructions that execute when the function is called.

**Example:**

```python
def display_investigation_message():
    print("investigate activity")
```

Important:

* The body must be **indented**
* Indentation separates the function's code from the rest of the program

---

## Calling a Function

After a function is defined, it can be executed by calling it.

**Syntax:**

```python
function_name()
```

**Example:**

```python
display_investigation_message()
```

**Output:**

```text
investigate activity
```

---

## Example in Cybersecurity

```python
def display_investigation_message():
    print("investigate activity")

application_status = "potential concern"
email_status = "okay"

if application_status == "potential concern":
    print("application_log:")
    display_investigation_message()

if email_status == "potential concern":
    print("email_log:")
    display_investigation_message()
```

### Output

```text
application_log:
investigate activity
```

### Explanation

* The function is defined once.
* It is called whenever suspicious activity is detected.
* `application_status` meets the condition, so the function runs.
* `email_status` does not meet the condition, so the function is not called.

---

## Advantages of Functions

* Reduce code duplication
* Improve readability
* Simplify maintenance
* Enable code reuse
* Help automate repetitive cybersecurity tasks

---

## Infinite Loops in Functions

Be careful when a function calls itself without a stopping condition.

**Example:**

```python
def func1():
    func1()
```

### What Happens?

1. `func1()` is called.
2. Inside the function, `func1()` calls itself again.
3. This continues indefinitely.
4. The program eventually crashes with a recursion error.

This is known as **infinite recursion**.

To avoid this, recursive functions must include a condition that stops further function calls.

# Working with Variables in Functions (Rewritten Notes)

## Introduction

Functions often need data to perform tasks. In Python, this data is passed into functions using **parameters** and **arguments**. Functions can also send data back to the program using **return statements**.

Understanding these concepts is important when writing reusable and efficient code, especially in cybersecurity tasks such as log analysis, user authentication, and security monitoring.

---

# Parameters

A **parameter** is a variable declared in a function definition. It acts as a placeholder that receives data when the function is executed.

### Example

```python
def calculate_remaining_attempts(max_attempts, used_attempts):
    print(max_attempts - used_attempts)
```

In this example:

* `max_attempts` is a parameter
* `used_attempts` is a parameter

These variables are available only within the function and are used to perform calculations.

---

# Arguments

An **argument** is the actual value supplied to a function when it is called.

### Example

```python
calculate_remaining_attempts(5, 2)
```

Here:

* `5` is passed to `max_attempts`
* `2` is passed to `used_attempts`

Python matches arguments to parameters based on their position unless specified otherwise.

### Visualization

```text
Function Definition:
max_attempts      used_attempts
      ↑                 ↑

Function Call:
      5                 2
```

---

# Return Statements

Functions can send information back to the code that called them using the `return` keyword.

### Example

```python
def calculate_remaining_attempts(max_attempts, used_attempts):
    return max_attempts - used_attempts
```

Instead of displaying the result, this function returns it.

### Important Points

* `return` ends the function immediately.
* Any code after a `return` statement will not execute.
* Returned values can be stored in variables and reused elsewhere.

---

# Using Returned Values

```python
def calculate_remaining_attempts(max_attempts, used_attempts):
    return max_attempts - used_attempts

remaining = calculate_remaining_attempts(3, 3)

if remaining <= 0:
    print("Account locked")
```

### Output

```text
Account locked
```

### Explanation

1. The function calculates `3 - 3`.
2. The result `0` is returned.
3. The returned value is stored in `remaining`.
4. Since `remaining` is not greater than zero, the account is considered locked.

A **global variable** is declared outside any function and can be accessed from most parts of a program.

### Example

This variable can be used in functions, loops, conditionals, and other sections of the program.

### Output


The function can access the global variable even though it was not created inside the function.

# Local Variables

A **local variable** exists only inside the function where it is created.

### Example

```python

```

Local variables in this function:

* `name` (parameter)

# Scope of Local Variables

```python

    greeting = "Welcome " + name
    return greeting

print(greeting)
```

### Result

```text
NameError
```

The error occurs because `greeting` only exists inside the function.

Once the function finishes execution, Python removes its local variables from memory.

---

# Global vs Local Variables

Consider the following example:

```python
user = "alice"

print("Before:", user)

def change_user():
    user = "bob"
    print("Inside:", user)

change_user()

print("After:", user)
```

### Output

```text

Inside: bob
After: alice
```

### Explanation

* The variable inside the function is local.
* It does not change the global variable.
* Python treats them as two separate variables even though they share the same name.

# Python Built-in Functions: `max()`, `min()`, and `sorted()` (Rewritten Notes)

## Introduction

Python provides several built-in functions that simplify data analysis and manipulation. Three commonly used functions are:

# `max()` Function

The `max()` function returns the highest value from a collection of items.

### Example

```python
session_times = [12, 2, 32, 19, 57, 22, 14]

print(max(session_times))
```

### Output

```text
57
```

### Explanation

The function examines all values in the list and returns the largest one.

### Cybersecurity Example

A security analyst might use `max()` to determine:

* Longest user session
* Highest number of failed login attempts
* Largest file transfer
* Maximum network latency

# `min()` Function

The `min()` function returns the smallest value from a collection of items.

### Example

```python
session_times = [12, 2, 32, 19, 57, 22, 14]

print(min(session_times))
```

### Output

```text
2
```

### Explanation

The function searches through the data and returns the lowest value.

### Cybersecurity Example

A security analyst might use `min()` to identify:

* Shortest login session
* Lowest response time
* Smallest data transfer
* Minimum number of login attempts

---

# Using `max()` and `min()` Together


### Output


# `sorted()` Function

The `sorted()` function arranges elements in a specific order and returns a new sorted list.

### Example

```python
session_times = [12, 2, 32, 19, 57, 22, 14]

print(sorted(session_times))
```

### Output

```text
[2, 12, 14, 19, 22, 32, 57]
```

### Explanation

By default, numbers are sorted from smallest to largest.

---

# Sorting Does Not Modify the Original Data

A common misconception is that `sorted()` changes the original list.

### Example

```python

print(sorted(session_times))
print(session_times)
```

### Output

```text
[2, 12, 14, 19, 22, 32, 57]
[12, 2, 32, 19, 57, 22, 14]
```

### Explanation

* `sorted()` creates a new sorted list.
* The original list remains unchanged.

Each character is sorted alphabetically and returned as a list.

# Data Type Limitation

The `sorted()` function requires comparable data types.

### Valid Example


### Invalid Example

### Result

```text

```

### Why?

Python cannot determine how to compare numbers and strings for sorting.

### Working with string data in a security context

In cybersecurity work, one of the most frequently used data types in Python is the **string**. A string is a sequence of characters arranged in order. It is used whenever data is not meant for mathematical operations like addition or subtraction. Examples in security environments include usernames, IP addresses, URLs, device IDs, and employee identifiers.

Analysts often need to inspect, extract, or validate parts of these strings. For instance, you may want to pull a segment from an IP address or check whether a username follows a required naming rule.

---

## Working with string positions (indices)

### What is an index?

An **index** is a numerical position assigned to each character in a string. It helps you locate specific characters.

Python starts counting from **0**. For example, consider the string `"h32rb17"`:

* h → 0
* 3 → 1
* 2 → 2
* r → 3
* b → 4
* 1 → 5
* 7 → 6

You can also use **negative indexing**, which counts from the end of the string:

* 7 → -1
* 1 → -2
* b → -3
* r → -4
* 2 → -5
* 3 → -6
* h → -7

---

## Extracting data using bracket notation

Bracket notation uses square brackets to access parts of a string based on index positions. It is commonly used in log analysis and data parsing.

For example:

```python
"h32rb17"[0]
```

This returns `"h"`.

If the string is stored in a variable:

```python
device_id = "h32rb17"
device_id[0]
```

Both expressions return the first character.

---

## Slicing strings

You can extract multiple characters using slicing. This is useful when only a portion of a value is needed, such as part of an IP address or URL segment.

Example:

```python
print("h32rb17"[0:3])
```

This returns `"h32"`.

Important detail: the start index is included, but the end index is excluded. So the slice stops just before the ending position.

---

## Useful string functions and methods

### Converting and measuring strings

* **str()** converts values into string format.
  Example:

  ```python
  string_id = str(19329302)
  ```

* **len()** returns the total number of characters in a string.
  Example use case: verifying a device ID format.

```python
if len("h32rb17") == 7:
    print("Valid device ID length")
```

---

### Changing letter case

* **.upper()** converts all characters to uppercase:

  ```python
  "Information Technology".upper()
  ```

  Output: `"INFORMATION TECHNOLOGY"`

* **.lower()** converts all characters to lowercase:

  ```python
  "Information Technology".lower()
  ```

  Output: `"information technology"`

---

### Finding positions with .index()

The **.index()** method returns the position of the first match of a character or substring.

Example:

```python
print("h32rb17".index("r"))
```

Output: `3`

If the character is not present, Python returns an error.

When duplicates exist, only the first occurrence is returned:

```python
print("r45rt46".index("r"))
```

Output: `0`

---

## Searching for substrings

A **substring** is a smaller sequence inside a larger string. For example, `"llo"` is part of `"hello"`.

The `.index()` method can also locate substrings:

```python
"tsnow, tshah, bmoreno - updated".index("tshah")
```

This returns `7`, which is the starting position of the substring.

Be careful when searching for partial matches. Searching for a short pattern like `"ts"` may return an earlier match that appears in a different word, rather than the intended one.

---

In cybersecurity analysis, these string operations are essential for extracting meaningful patterns from logs, validating identifiers, and locating relevant security data efficiently.

### Basics of Regular Expressions (Regex)

A **regular expression (regex)** is a sequence of characters used to define a search pattern. In Python, regex is commonly used to locate structured patterns inside text, such as IP addresses, email addresses, usernames, or device identifiers.

To use regular expressions in Python, you first need to import the **re module**:

```python
import re
```

Regex patterns are written as strings and then passed into functions from the `re` module to search within other strings.

One of the most commonly used functions is **`re.findall()`**, which returns a list of all matches for a given pattern. It takes two arguments:

1. The regex pattern
2. The string to search in

Example:

```python id="r1a9kx"
import re
re.findall("ts", "tsnow, tshah, bmoreno")
```

Output:

```
['ts', 'ts']
```

This shows that both occurrences of `"ts"` were found in the text.

---

## Special symbols in regular expressions

Regex becomes more powerful when you use **special character symbols** instead of only literal text.

### Character class symbols

These symbols match specific types of characters:

* `\w` → Matches any alphanumeric character (A–Z, a–z, 0–9) and underscore `_`
* `\d` → Matches any digit (0–9)
* `\s` → Matches whitespace (spaces, tabs, new lines)
* `.` → Matches any character except newline
* `\.` → Matches a literal period (`.`), since `.` alone is a special wildcard

Example using `\w`:

```python id="q9k2lm"
import re
re.findall("\w", "h32rb17")
```

Output:

```
['h', '3', '2', 'r', 'b', '1', '7']
```

Example using `\d`:

```python id="x8p1va"
import re
re.findall("\d", "h32rb17")
```

Output:

```
['3', '2', '1', '7']
```

---

## Quantifiers (controlling repetition)

Quantifiers define **how many times a pattern should appear**.

### Common quantifiers:

* `+` → One or more occurrences
* `*` → Zero or more occurrences
* `{n}` → Exactly n occurrences
* `{n,m}` → Between n and m occurrences

---

### Using `+` (one or more matches)

```python id="m2n8dp"
import re
re.findall("\d+", "h32rb17")
```

Output:

```
['32', '17']
```

This groups consecutive digits together instead of returning them one by one.

---

### Using `*` (zero or more matches)

```python id="t7q4xz"
import re
re.findall("\d*", "h32rb17")
```

This returns many matches, including empty strings, because it also matches positions where no digits exist.

---

### Using `{n}` (exact repetitions)

```python id="c5w8hf"
import re
re.findall("\d{2}", "h32rb17 k825t0m c2994eh")
```

This finds only sequences of exactly two digits.

Important behavior: if a longer sequence exists (like `123`), the pattern matches step-by-step (e.g., `12`, then `23` depending on scanning).

---

### Using `{n,m}` (range of repetitions)

```python id="p9d3ka"
import re
re.findall("\d{1,3}", "h32rb17 k825t0m c2994eh")
```

This returns numbers with 1 to 3 digits, such as `3`, `32`, `825`, etc.

---

## Building a regex pattern

Creating regex patterns involves breaking a task into small parts and combining symbols.

Example scenario: extracting usernames and login attempts from a log string.

```python
employee_logins_string = "1001 bmoreno: 12 Marketing 1002 tshah: 7 Human Resources 1003 sgilmore: 5 Finance"
```

We want:

* Username
* Number of login attempts
  But not the employee ID or department.

### Pattern breakdown:

* `\w+` → username (word characters)
* `:` → literal colon
* `\s` → space
* `\d+` → number of logins

Final regex:

```python id="v3k9rt"
import re

pattern = "\w+:\s\d+"
employee_logins_string = "1001 bmoreno: 12 Marketing 1002 tshah: 7 Human Resources 1003 sgilmore: 5 Finance"

print(re.findall(pattern, employee_logins_string))
```

This extracts:

```
['bmoreno: 12', 'tshah: 7', 'sgilmore: 5']
```

---

## Important note

Regex is extremely powerful but must be used carefully. Poorly designed patterns may:

* Return extra unwanted matches
* Miss important data
* Produce misleading results

Testing and refining patterns is essential in cybersecurity work, especially when analyzing logs or detecting patterns in large datasets.
